Question 1
Confusion Matrix → Accuracy  |  4 marks  |  CLO3
A spam-email filter (class 1 = spam). Confusion matrix: TP = 80, FN = 20, FP = 40, TN = 60.
Accuracy = (TP + TN) / (TP + TN + FP + FN)

Given Values (from question)

All four values are given directly in the confusion matrix
TP = 80   (correctly predicted spam) FN = 20   (spam predicted as not-spam — missed) FP = 40   (not-spam predicted as spam — false alarm) TN = 60   (correctly predicted not-spam)

(a) Formula (1 mark)

Step 1: State the accuracy formula
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Accuracy = fraction of ALL predictions that are correct (both classes).

(b) Compute Accuracy (2 marks)

Step 2: Substitute given values
Numerator (correct):  TP + TN = 80 + 60 = 140 Denominator (total): TP + TN + FP + FN = 80 + 60 + 40 + 20 = 200 Accuracy = 140 / 200 = 0.7000

(c) Why accuracy can be misleading (1 mark)

Answer
When one class is much rarer (imbalanced), a model can achieve high accuracy by always predicting the majority class, despite having no real ability to identify the minority (rare) class.
Question 2
Regression Error → RMSE  |  4 marks  |  CLO3
House prices ($1,000s): y = [10, 14, 8, 12, 6], predicted ŷ = [12, 11, 9, 15, 5]
RMSE = √[(1/n) Σ (ŷ₁ − y₁)²]

Given Values (from question table)

Five data pairs from the table
n = 5 (number of properties) Property: 1 2 3 4 5 Actual y: 10 14 8 12 6 ← given Predicted: 12 11 9 15 5 ← given

(a) Residuals (1 mark)

Step 1: Compute each residual = ŷ₁ − y₁
Property 1: 12 − 10 = 2 Property 2: 11 − 14 = −3 Property 3: 9 − 8 = 1 Property 4: 15 − 12 = 3 Property 5: 5 − 6 = −1
Each residual = predicted minus actual. Positive = overpredicted, negative = underpredicted.

(b) MSE (1 mark)

Step 2: Square each residual, sum, divide by n
Squared residuals: 2² = 4, (-3)² = 9, 1² = 1, 3² = 9, (-1)² = 1 Sum of squares = 4 + 9 + 1 + 9 + 1 = 24 MSE = 24 / 5 = 4.8000

(c) RMSE (2 marks)

Step 3: Take square root of MSE
RMSE = √MSE = √4.8 = 2.1909
RMSE is in the same units as y (thousands of dollars). On average, predictions are off by about $2,191.
Question 3
Regression Error → MAE  |  3 marks  |  CLO3
Same data as Q2: y = [10, 14, 8, 12, 6], ŷ = [12, 11, 9, 15, 5]
MAE = (1/n) Σ |ŷ₁ − y₁|

Given Values (from Q2 — same data)

Residuals already computed in Q2
Residuals (ŷ − y): [+2, −3, +1, +3, −1] n = 5

(a) Absolute Errors (1 mark)

Step 1: Take absolute value of each residual
|+2| = 2 |−3| = 3 |+1| = 1 |+3| = 3 |−1| = 1

(b) MAE (1 mark)

Step 2: Average the absolute errors
Sum = 2 + 3 + 1 + 3 + 1 = 10 MAE = 10 / 5 = 2.0000

(c) Why MAE is less sensitive (1 mark)

Answer
MAE treats all errors linearly (absolute value), whereas RMSE squares errors first, which disproportionately amplifies the contribution of large errors (outliers).

Question 5
Classification Loss → Binary Cross-Entropy  |  5 marks  |  CLO2
4 samples. p̂ = [0.9, 0.2, 0.6, 0.4], y = [1, 0, 1, 0]. Hints: ln(0.9) ≈ −0.1054, ln(0.8) ≈ −0.2231, ln(0.6) ≈ −0.5108
BCE = −[y·ln(p̂) + (1−y)·ln(1−p̂)]

Given Values (from question table + hints)

Four predicted probabilities and four true labels
Sample: 1 2 3 4 Predicted p̂: 0.9 0.2 0.6 0.4 ← given True label y: 1 0 1 0 ← given Given log values from hints: ln(0.9) = −0.1054 ln(0.8) = −0.2231 (this is ln(1 − 0.2), needed for sample 2) ln(0.6) = −0.5108 ln(0.4) = −0.9163 (derived: ln(1 − 0.6), needed for sample 3) ln(0.1) = −2.3026 (derived: ln(1 − 0.9), needed for sample 1) Note: We need ln(1−p̂) for each sample too. These are computed as: Sample 1: 1−0.9 = 0.1, so ln(0.1) = −2.3026 Sample 2: 1−0.2 = 0.8, so ln(0.8) = −0.2231 (given in hint) Sample 3: 1−0.6 = 0.4, so ln(0.4) = −0.9163 Sample 4: 1−0.4 = 0.6, so ln(0.6) = −0.5108 (given in hint)

(a) Per-sample BCE formula (1 mark)

Step 1: State the per-sample loss term
L₁ = −[ y₁ · ln(p̂₁) + (1 − y₁) · ln(1 − p̂₁) ]
When y=1 the first term is active; when y=0 the second term is active.

(b) Loss for each sample (2 marks)

Step 2: Apply formula to each sample
Sample 1 (y=1, p̂=0.9): L₁ = −[ 1 × ln(0.9) + 0 × ln(0.1) ] = −[ 1 × (−0.1054) + 0 ] = −(−0.1054) = 0.1054 Sample 2 (y=0, p̂=0.2): L₂ = −[ 0 × ln(0.2) + 1 × ln(0.8) ] = −[ 0 + 1 × (−0.2231) ] = −(−0.2231) = 0.2231 Sample 3 (y=1, p̂=0.6): L₃ = −[ 1 × ln(0.6) + 0 × ln(0.4) ] = −[ 1 × (−0.5108) + 0 ] = −(−0.5108) = 0.5108 Sample 4 (y=0, p̂=0.4): L₄ = −[ 0 × ln(0.4) + 1 × ln(0.6) ] = −[ 0 + 1 × (−0.5108) ] = −(−0.5108) = 0.5108

(c) Mean BCE (2 marks)

Step 3: Average the four loss values
Sum = 0.1054 + 0.2231 + 0.5108 + 0.5108 = 1.3501 Mean BCE = 1.3501 / 4 = 0.3375
Question 6
Decision Tree → Gini Impurity  |  3 marks  |  CLO2
Node has 10 samples: 7 class A, 3 class B.
Gini = 1 − Σ p₁²

Given Values (from question)

Counts given directly
Total samples = 10 Class A count = 7 ← given Class B count = 3 ← given

(a) Class proportions (1 mark)

Step 1: Divide each count by total
pₖ = 7 / 10 = 0.7 pₙ = 3 / 10 = 0.3

(b) Gini impurity (2 marks)

Step 2: Apply the Gini formula
Gini = 1 − (pₖ² + pₙ²) = 1 − (0.7² + 0.3²) = 1 − (0.49 + 0.09) = 1 − 0.58 = 0.4200
Gini = 0 means pure node (all one class). Gini = 0.5 means maximum impurity for 2 classes (50/50 split).
Question 7
Decision Tree → Entropy  |  4 marks  |  CLO2
Binary node: 10 samples, 2 positive, 8 negative. Hints: log₂(0.2) ≈ −2.3219, log₂(0.8) ≈ −0.3219
H = − Σ p₁ · log₂(p₁)

Given Values (from question + hints)

Counts and log values given
Total samples = 10 Positive count = 2 ← given Negative count = 8 ← given log₂(0.2) = −2.3219 ← given in hint log₂(0.8) = −0.3219 ← given in hint

(a) Class proportions (1 mark)

Step 1: Compute proportions
p₉ = 2/10 = 0.2 p₊ = 8/10 = 0.8

(b) Entropy (2 marks)

Step 2: Apply entropy formula using hint values
H = −[ p₉ · log₂(p₉) + p₊ · log₂(p₊) ] = −[ 0.2 × (−2.3219) + 0.8 × (−0.3219) ] = −[ −0.4644 + (−0.2575) ] = −[ −0.7219 ] = 0.7219 bits

(c) Maximum entropy (1 mark)

Answer
Maximum entropy for a binary node = 1.0 bit, achieved when the split is perfectly balanced: 50% positive, 50% negative.
Question 8
Confusion Matrix → F1-Score  |  5 marks  |  CLO3
Medical screening (class 1 = disease). TP = 40, FN = 10, FP = 30, TN = 20.
Precision = TP/(TP+FP)  |  Recall = TP/(TP+FN)  |  F1 = 2PR/(P+R)

Given Values (from confusion matrix)

All values from the matrix
TP = 40 ← given FN = 10 ← given FP = 30 ← given TN = 20 ← given

(a) Precision (1 mark)

Step 1: Precision = TP / (TP + FP)
Precision = 40 / (40 + 30) = 40 / 70 = 0.5714
Of all predicted positives, 57.14% are truly positive.

(b) Recall (1 mark)

Step 2: Recall = TP / (TP + FN)
Recall = 40 / (40 + 10) = 40 / 50 = 0.8000
Of all actual positives, 80% are correctly identified.

(c) F1-score (2 marks)

Step 3: F1 = 2 · Precision · Recall / (Precision + Recall)
F1 = 2 × (0.5714 × 0.8000) / (0.5714 + 0.8000) = 2 × 0.4571 / 1.3714 = 0.9143 / 1.3714 = 0.6667

(d) Why harmonic mean? (1 mark)

Answer
The harmonic mean is always closer to the smaller of the two values, so F1 penalises models where either precision or recall is very low — the arithmetic mean could mask a poor score on one metric.
Question 9
Gradient Descent → Single Feature  |  6 marks  |  CLO2
ŷ = w₀ + w₁x, w₀ = 0, w₁ = 0, α = 0.1. Data: x = [1, 2, 4], y = [2, 5, 9]
∂J/∂w₁ = (2/n) Σ (ŷ₁ − y₁) · x₁  |  w₁ ← w₁ − α · ∂J/∂w₁

Given Values (from question)

Weights, learning rate, and training data all given
w₀ = 0 ← given (initial weight) w₁ = 0 ← given (initial weight) α = 0.1 ← given (learning rate) n = 3 ← given (3 data points) Data (given): x: [1, 2, 4] y: [2, 5, 9]

(a) Initial predictions (1 mark)

Step 1: ŷ₁ = w₀ + w₁ · x₁ with w₀=0, w₁=0
Point 1: ŷ₁ = 0 + 0 × 1 = 0 Point 2: ŷ₂ = 0 + 0 × 2 = 0 Point 3: ŷ₃ = 0 + 0 × 4 = 0 All predictions are 0 (initial weights are zero).

(b) Residuals (1 mark)

Step 2: Residual = ŷ₁ − y₁
Point 1: 0 − 2 = −2 Point 2: 0 − 5 = −5 Point 3: 0 − 9 = −9

(c) Gradient (2 marks)

Step 3: Compute ∂J/∂w₁ = (2/n) Σ (ŷ₁ − y₁) · x₁
Multiply each residual by its x value: (−2) × 1 = −2 (−5) × 2 = −10 (−9) × 4 = −36 Sum = (−2) + (−10) + (−36) = −48 ∂J/∂w₁ = (2/3) × (−48) = 2 × (−16) = −32.0000

(d) Weight update (2 marks)

Step 4: Apply update rule
w₁ ← w₁ − α × ∂J/∂w₁ = 0 − 0.1 × (−32.0000) = 0 + 3.2000 = 3.2000
The negative gradient times negative learning rate = positive update, meaning w₁ increases to reduce the underpredictions.
Question 10
Gradient Descent → Two Features  |  6 marks  |  CLO2
w₀=0.2, w₁=0.5, w₂=0.1. Sample: x₁=3, x₂=2, y=5.0, α=0.1. Per-sample gradient: ∂J/∂w₏ = 2(ŷ − y) · x₏
ŷ = w₀ + w₁x₁ + w₂x₂

Given Values (from question)

Weights, sample, and learning rate all given
w₀ = 0.2 ← given w₁ = 0.5 ← given w₂ = 0.1 ← given x₁ = 3 ← given x₂ = 2 ← given y = 5.0 ← given α = 0.1 ← given

(a) Prediction (1 mark)

Step 1: ŷ = w₀ + w₁x₁ + w₂x₂
ŷ = 0.2 + 0.5 × 3 + 0.1 × 2 = 0.2 + 1.5 + 0.2 = 1.9000

(b) Error (1 mark)

Step 2: Error = ŷ − y
Error = 1.9 − 5.0 = −3.1000
Negative error means underprediction.

(c) Gradient ∂J/∂w₂ (2 marks)

Step 3: Use per-sample gradient formula with x₂
∂J/∂w₂ = 2 × (ŷ − y) × x₂ = 2 × (−3.1) × 2 = 2 × (−6.2) = −12.4000

(d) Update w₂ (2 marks)

Step 4: Apply update rule
w₂ ← w₂ − α × ∂J/∂w₂ = 0.1 − 0.1 × (−12.4000) = 0.1 + 1.2400 = 1.3400
w₂ increases substantially because the model underpredicts and x₂ contributes to the error.
Question 11

Fraud detection (class 1 = fraud). TP = 60, FN = 40, FP = 15, TN = 85.
Precision = TP/(TP+FP)  |  Recall = TP/(TP+FN)

Given Values (from confusion matrix)

All values from the matrix
TP = 60 ← given FN = 40 ← given FP = 15 ← given TN = 85 ← given

(a) Precision (1.5 marks)

Step 1: Precision = TP / (TP + FP)

(b) Recall (1.5 marks)

Question 12
Regression → R²  |  6 marks  |  CLO3
5 samples. y = [3, 5, 7, 9, 11], ŷ = [4, 5, 6, 10, 10]
R² = 1 − (SSₒ / SSᵢₓ)  |  SSₒ = Σ(y₁−ŷ₁)²  |  SSᵢₓ = Σ(y₁−ȳ)²

Given Values (from question table)

Five actual/predicted pairs from the table
Sample: 1 2 3 4 5 y: 3 5 7 9 11 ← given ŷ: 4 5 6 10 10 ← given n = 5

(a) Mean ȳ (1 mark)

Step 1: Average of actual values
ȳ = (3 + 5 + 7 + 9 + 11) / 5 = 35 / 5 = 7.0000

(b) SSₒ (Residual Sum of Squares) (2 marks)

Step 2: Compute (y₁ − ŷ₁)² for each sample
Sample 1: (3 − 4)² = (−1)² = 1 Sample 2: (5 − 5)² = ( 0)² = 0 Sample 3: (7 − 6)² = ( 1)² = 1 Sample 4: (9 − 10)² = (−1)² = 1 Sample 5: (11 − 10)² = ( 1)² = 1 SS_res = 1 + 0 + 1 + 1 + 1 = 4

(c) SSᵢₓ (Total Sum of Squares) (1 mark)

Step 3: Compute (y₁ − ȳ)² for each sample
Sample 1: (3 − 7)² = (−4)² = 16 Sample 2: (5 − 7)² = (−2)² = 4 Sample 3: (7 − 7)² = ( 0)² = 0 Sample 4: (9 − 7)² = ( 2)² = 4 Sample 5: (11 − 7)² = ( 4)² = 16 SS_tot = 16 + 4 + 0 + 4 + 16 = 40

(d) R² (2 marks)

Step 4: Apply R² formula
R² = 1 − (SS_res / SS_tot) = 1 − (4 / 40) = 1 − 0.1000 = 0.9000
R² = 0.9000 means the model explains 90% of the variance in the actual values.
Question 13

3-class network. Logit vector z = [2.0, 1.0, 0.0]. Hints: e² ≈ 7.3891, e¹ ≈ 2.7183, e&sup0; = 1.0000

Logit vector and exponential values all given

Step 1: State the general formula
softmax(zᵢ) = e^zᵢ / (e^z₁ + e^z₂ + e^z₃)

Step 2: Compute each eⁿ₁ using the hints
e^z₁ = e^2.0 = 7.3891 (from hint) e^z₂ = e^1.0 = 2.7183 (from hint) e^z₃ = e^0.0 = 1.0000 (from hint) Sum = 7.3891 + 2.7183 + 1.0000 = 11.1074

(c) Softmax probabilities (2 marks)

Step 3: Divide each exponential by the sum
Class 1: 7.3891 / 11.1074 = 0.6653 Class 2: 2.7183 / 11.1074 = 0.2448 Class 3: 1.0000 / 11.1074 = 0.0900 Verification: 0.6653 + 0.2448 + 0.0900 = 0.9999 ≈ 1.0000 ✓ (minor rounding; mathematically exact sum = 1.0)

Question 15
Decision Tree → Information Gain  |  8 marks  |  CLO2
Parent: 10 samples (4 positive, 6 negative). Split into Left (5: 1 pos, 4 neg) and Right (5: 3 pos, 2 neg). Hints: log₂(0.2)≈−2.3219, log₂(0.8)≈−0.3219, log₂(0.4)≈−1.3219, log₂(0.6)≈−0.7370
IG = H(parent) − Σ (|child|/|parent|) · H(child)

Given Values (from question table + hints)

All sample counts from the table; log values from hints
Parent: 10 samples total — 4 positive, 6 negative ← given Left child: 5 samples — 1 positive, 4 negative ← given Right child: 5 samples — 3 positive, 2 negative ← given Given log values: log₂(0.2) = −2.3219 ← hint (needed for: 1/5=0.2, 4/5=0.8 on left) log₂(0.8) = −0.3219 ← hint log₂(0.4) = −1.3219 ← hint (needed for: 2/5=0.4, 3/5=0.6 on right) log₂(0.6) = −0.7370 ← hint

(a) Parent entropy H(parent) (2 marks)

Step 1: Compute parent class proportions then entropy
Parent proportions: p_pos = 4/10 = 0.4 p_neg = 6/10 = 0.6 H(parent) = −[ 0.4 × log₂(0.4) + 0.6 × log₂(0.6) ] = −[ 0.4 × (−1.3219) + 0.6 × (−0.7370) ] = −[ −0.5288 + (−0.4422) ] = −[ −0.9710 ] = 0.9710 bits

(b) Child entropies H(left) and H(right) (3 marks)

Step 2a: Left child — 1 positive, 4 negative out of 5
Left proportions: p_pos = 1/5 = 0.2 p_neg = 4/5 = 0.8 H(left) = −[ 0.2 × log₂(0.2) + 0.8 × log₂(0.8) ] = −[ 0.2 × (−2.3219) + 0.8 × (−0.3219) ] = −[ −0.4644 + (−0.2575) ] = −[ −0.7219 ] = 0.7219 bits
Step 2b: Right child — 3 positive, 2 negative out of 5
Right proportions: p_pos = 3/5 = 0.6 p_neg = 2/5 = 0.4 H(right) = −[ 0.6 × log₂(0.6) + 0.4 × log₂(0.4) ] = −[ 0.6 × (−0.7370) + 0.4 × (−1.3219) ] = −[ −0.4422 + (−0.5288) ] = −[ −0.9710 ] = 0.9710 bits

(c) Weighted average entropy (1 mark)

Step 3: Weight each child by its proportion of parent samples
Weight of left = |left| / |parent| = 5/10 = 0.5 Weight of right = |right| / |parent| = 5/10 = 0.5 Weighted H = 0.5 × H(left) + 0.5 × H(right) = 0.5 × 0.7219 + 0.5 × 0.9710 = 0.3610 + 0.4855 = 0.8465 bits

(d) Information Gain (2 marks)

Step 4: IG = H(parent) − weighted average H(children)
IG = H(parent) − Weighted H(children) = 0.9710 − 0.8465 = 0.1245 bits
The split reduces entropy by 0.1245 bits. A higher IG means a better split — this is the value used to choose the best feature at each node.
