Topic 1: Introduction to Machine Learning

TTTC2453 — The foundational concepts that everything else in this course builds upon.

🔄

The Paradigm Shift

For decades, if you wanted a computer to do something, you had to tell it exactly what to do, step by step. That is traditional programming: a human writes explicit rules, the computer follows them, and out comes the answer.

Traditional Programming INPUT + RULES → OUTPUT

For example, if you wanted to classify emails as spam or not, you would write rules like: "If the subject contains 'FREE MONEY', mark as spam. If the sender is in contacts, mark as not spam." The problem? Real life is messy. Spammers change their wording. New patterns emerge. You would spend forever writing and updating rules.

Machine Learning (Core Idea)

Instead of writing rules yourself, you give the computer examples (data) and it discovers the rules on its own. You flip the equation around.

Machine Learning INPUT + OUTPUT (labeled examples) → RULES (learned model)

So with spam: you show the computer thousands of emails that are already labeled "spam" or "not spam," and it figures out what patterns distinguish them — patterns no human would think to write rules for.

Analogy: Learning to Cook

Traditional programming is like following a recipe exactly: "Add 2 tsp salt, bake at 180°C for 25 minutes." The recipe is the rules; the dish is the output.

Machine learning is like a chef who tastes hundreds of dishes, notices patterns ("salty dishes usually had this much salt relative to the meat"), and develops an intuition for what makes a dish good. Nobody wrote the recipe — the chef learned it from examples.

Mapping: The hundreds of dishes tasted = training data. The chef's developing intuition = the learned model. Tasting a new dish and rating it = making a prediction.

Key Takeaway

Traditional programming: you write the rules. Machine learning: the computer learns the rules from data. This is the single most important conceptual shift in modern computing.

💡

Why ML Now?

Machine learning as a concept has existed since the 1950s. So why did it explode in the last decade? Three things came together at the right time:

1. Data Abundance

The world generates 2.5 quintillion bytes of data every day. Every social media post, online purchase, GPS ping, and medical scan adds to the pile. ML algorithms are hungry for data — the more they see, the better they learn. Without big data, ML would still be a curiosity. With it, it is a superpower.

2. Compute Power

Training an ML model means doing millions (or billions) of matrix multiplications. In the 1990s this took weeks on a single computer. Today, GPUs (originally designed for rendering video games) can do thousands of these calculations in parallel. Cloud computing means anyone can rent a powerful GPU by the hour for a few dollars.

3. Algorithmic Maturity

Key breakthroughs — like backpropagation (1986), support vector machines (1990s), and deep learning (2012, AlexNet) — gave us algorithms that actually work. Libraries like scikit-learn, TensorFlow, and PyTorch made these algorithms accessible to anyone who can write a few lines of Python.

Analogy: The Perfect Storm

Think of ML like a rocket. Data is the fuel. Compute power is the engine. Algorithms are the navigation system. All three existed separately for decades, but only when they converged did ML take off.

Mapping: Fuel = data (you need enough of it). Engine = GPU/cloud compute (raw power to process it). Navigation = mature algorithms (methods that actually work well).

Key Takeaway

ML is not new — but the convergence of massive data, powerful hardware, and proven algorithms made it practical for the first time in the 2010s. That is why this course exists now and not 20 years ago.

The Hierarchy: AI > ML > DL

People often use AI, ML, and deep learning interchangeably, but they are nested categories, like Russian dolls:

Level What It Is Example
Artificial Intelligence (AI) The broadest umbrella: any technique that enables machines to mimic human intelligence. Expert systems, search algorithms, robotics, ML, logic-based AI
Machine Learning (ML) A subset of AI: systems that learn from data rather than following hard-coded rules. Linear regression, decision trees, random forests, K-means
Deep Learning (DL) A subset of ML: models with many layers of artificial neurons (deep neural networks). CNNs for images, RNNs for text, Transformers for everything
Common Misconception

Not all AI is machine learning. A chess engine that uses a brute-force search tree is AI but not ML — it has no learning from data. Similarly, not all ML is deep learning. A decision tree is ML but not DL.

Analogy: Vehicles

AI = all vehicles. ML = cars (a specific type of vehicle). DL = electric cars (a specific type of car). Every electric car is a car, and every car is a vehicle — but not every vehicle is a car.

Mapping: Vehicles = AI (broadest). Cars = ML (learn from data). Electric cars = DL (use deep neural networks).

Key Takeaway

AI is the big picture (machines acting intelligently). ML is the main tool within AI that uses data to learn. DL is the current state-of-the-art technique within ML, using layered neural networks. In this course, we focus primarily on ML, which gives us the tools to understand DL later.

Biological Neuron vs Artificial Neuron

Machine learning is loosely inspired by how the brain works. Let us look at both sides to understand the analogy and its limits.

The Biological Neuron

A neuron in your brain has three key parts:

  1. Dendrites — branches that receive electrical signals from other neurons.
  2. Cell body (soma) — processes the incoming signals. If the combined input exceeds a certain threshold, the neuron "fires."
  3. Axon — a long tail that transmits the output signal to the next neurons.

The Artificial Neuron (Perceptron)

In 1958, Frank Rosenblatt created the perceptron — a mathematical model inspired by the biological neuron:

  1. Inputs (X) — numerical values (like dendrites receiving signals).
  2. Weights (W) — each input is multiplied by a weight. A higher weight means that input matters more. This is like the synaptic strength of a biological connection.
  3. Bias (b) — an extra number added to shift the threshold. This is like how some neurons are naturally more likely to fire.
  4. Summation + Activation — all weighted inputs are summed up, and an activation function decides whether the neuron "fires" (produces an output).
Important Limitation

Artificial neurons are a very rough analogy to biological neurons. Real neurons are vastly more complex (chemical signals, dendrite trees, timing, plasticity). Do not take the analogy too literally — it is a useful starting point, not a faithful replica.

Analogy: A Voting Panel

Imagine 5 judges scoring a diving competition. Each judge gives a score (input). Some judges' opinions carry more weight (weight). The referee adds an adjustment factor to set the baseline (bias). The total is calculated, and if it passes a threshold, the diver gets a medal (activation fires).

Mapping: Judges' scores = inputs (X). Judge importance = weights (W). Baseline adjustment = bias (b). Threshold check = activation function. Medal decision = output.

Key Takeaway

An artificial neuron takes in numerical inputs, multiplies each by a weight, adds a bias, and passes the result through an activation function. It is a simplified mathematical model inspired by biology, but it is the fundamental building block of all neural networks.

🔧

Anatomy of a Perceptron

Let us formalize exactly what happens inside a single perceptron. This is one of the most important formulas in machine learning.

The Perceptron Formula z = w₁·x₁ + w₂·x₂ + w₃·x₃ + ... + wn·xn + b

Or more compactly using summation notation:

Compact Form z = Σ(wi · xi) + b   for i = 1 to n

Where:
xi = the i-th input feature
wi = the weight (importance) of the i-th input
b = bias (a constant offset)
z = the weighted sum (the "pre-activation" value)
n = the total number of input features

What Each Component Does

ComponentRoleBiological Equivalent
Inputs (x)Raw data values fed into the neuronSignals received by dendrites
Weights (w)Determine how important each input is. Higher weight = more influence.Synaptic strength of connections
Bias (b)Shifts the activation threshold. Allows the neuron to fire even if all inputs are zero.Intrinsic excitability of the neuron
Summation (Σ)Combines all weighted inputs + bias into one number (z)Integration at the cell body
Activation f(z)Decides the final output based on z. Determines if/when the neuron "fires."Firing threshold
Worked Example: A Perceptron with 3 Inputs
1 Define the inputs, weights, and bias
x₁ = 2, x₂ = 4, x₃ = -1
w₁ = 0.5, w₂ = -0.3, w₃ = 0.8
b = 1.0
2 Multiply each input by its weight
w₁·x₁ = 0.5 × 2 = 1.0
w₂·x₂ = -0.3 × 4 = -1.2
w₃·x₃ = 0.8 × (-1) = -0.8
3 Sum the weighted inputs and add bias
z = 1.0 + (-1.2) + (-0.8) + 1.0
z = 1.0 - 1.2 - 0.8 + 1.0
z = 0.0
4 Interpret the result
z = 0.0 means the neuron is exactly at the threshold.
With a step function (fires if z ≥ 0): output = 1
With a sigmoid: output = 1/(1+e^0) = 0.5
The negative weight on x₂ dragged the sum down — that feature reduces the neuron's output.
Plain-English Recap

The perceptron is a tiny decision-making unit. It takes in a bunch of numbers (inputs), multiplies each by how important it is (weight), adds a baseline adjustment (bias), sums everything up into one number (z), and then decides whether to "fire" or not (activation function). The magic is that during training, we adjust the weights and bias until the perceptron makes the right decisions. That learning process is the heart of machine learning.

⚡

Activation Functions

Without an activation function, a perceptron would just be a fancy linear equation: z = W·X + b. That can only draw straight lines. Activation functions introduce non-linearity, allowing the network to learn complex, curved decision boundaries.

1. Step Function (Heaviside)

The simplest activation. It outputs 0 or 1 — a hard yes/no.

Step Function f(z) = { 1 if z ≥ 0,   0 if z < 0 }

This is what Rosenblatt's original perceptron used. It works for simple binary decisions, but it is not differentiable (you cannot measure how much a tiny change in z affects the output), which makes it impossible to use with modern training algorithms like gradient descent.

2. Sigmoid Function

The sigmoid "squashes" any real number into a value between 0 and 1, making it perfect for probability-like outputs.

Sigmoid Function σ(z) = 1 / (1 + e-z)
Worked Example: Sigmoid Calculation
1 Suppose z = 2.0
σ(2.0) = 1 / (1 + e-2.0)
2 Calculate e-2.0
e-2.0 ≈ 0.1353
3 Compute the result
σ(2.0) = 1 / (1 + 0.1353) = 1 / 1.1353 ≈ 0.881
So the neuron outputs ~0.88, which we could interpret as 88% confidence.
4 Try a negative input: z = -3.0
e-(-3.0) = e3.0 ≈ 20.086
σ(-3.0) = 1 / (1 + 20.086) = 1 / 21.086 ≈ 0.047
Very low output — the neuron is "not firing."
Key Property of Sigmoid

When z is very large and positive, σ(z) approaches 1. When z is very large and negative, σ(z) approaches 0. When z = 0, σ(z) = 0.5 exactly. This S-shaped curve makes it behave like a smooth probability estimate.

Why Not Just Use Step?

The step function has a fatal flaw: it is not differentiable. To train a network, we need to know: "If I nudge this weight a tiny bit, how does the output change?" That requires a derivative. The sigmoid has a nice, smooth derivative everywhere, making gradient-based training possible. This is why sigmoid (and later ReLU, softmax, etc.) replaced the step function in modern ML.

Analogy: Light Switch vs Dimmer

The step function is like a light switch — on or off, nothing in between. The sigmoid is like a dimmer knob — you can smoothly go from dark (0) to bright (1), with every brightness level in between. The dimmer is more useful because you can fine-tune it, not just toggle.

Mapping: Light switch = step function (binary output). Dimmer knob = sigmoid (continuous output between 0 and 1). Fine-tuning = gradient descent adjusting weights smoothly.

Key Takeaway

Activation functions add non-linearity so neural networks can learn complex patterns. The step function is the simplest (on/off) but not trainable. The sigmoid squashes output to (0, 1) and is differentiable, making it a workhorse of early neural networks.

The Weight Update Loop

Here is how a perceptron actually learns. The process is beautifully simple:

  1. Make a prediction — Feed an input through the perceptron (compute z, apply activation). Get an output.
  2. Calculate the error — Compare the prediction to the correct answer (the "label"). Error = Actual - Predicted (or some loss function).
  3. Nudge the weights — If the prediction was too high, reduce the weights. If too low, increase them. The bigger the error, the bigger the nudge.
  4. Repeat — Do this for every training example, thousands of times, until the errors get small.
Weight Update Rule (Simplified) wnew = wold + η · error · xi

Where η (eta) is the learning rate — a small number like 0.01 that controls how big each nudge is. Too big = overshoot and never converge. Too small = learn painfully slowly.

Worked Example: One Weight Update
1 Setup: A perceptron with 1 input
x = 3.0, w = 0.2, b = 0.0
True label = 1 (this is a positive example)
2 Forward pass (prediction)
z = w · x + b = 0.2 × 3.0 + 0.0 = 0.6
Using step function: z ≥ 0, so prediction = 1
(Correct! No update needed in this case.)
3 Now try a negative example: x = -2.0, label = 0
z = 0.2 × (-2.0) + 0.0 = -0.4
Step function: z < 0, so prediction = 0
(Also correct!)
4 A wrong prediction: x = 2.0, label = 0, current w = 0.2
z = 0.2 × 2.0 = 0.4
Step function: z ≥ 0, so prediction = 1 (WRONG!)
error = actual - predicted = 0 - 1 = -1
Let η = 0.1
w_new = 0.2 + 0.1 × (-1) × 2.0
w_new = 0.2 - 0.2 = 0.0
The weight decreased! The perceptron learned that x=2 should NOT fire.
Analogy: Learning to Shoot Arrows

Imagine shooting an arrow at a target, but blindfolded. After each shot, someone tells you "too far left" or "too far right" (the error). You adjust your aim (update the weight) and shoot again. Over many shots, you zero in on the bullseye.

Mapping: Arrow shot = prediction. Target = actual label. Feedback ("too far left") = error. Adjusting aim = weight update. Blindfold = you don't know the rules; you learn from feedback. The learning rate = how aggressively you adjust after each shot.

Common Misconception

Students sometimes think the perceptron "understands" what it is doing. It does not. It is just iteratively adjusting numbers (weights) to minimize errors. The "intelligence" is in the data and the learning algorithm, not in any conscious understanding by the model.

Plain-English Recap

A perceptron learns by making predictions, measuring how wrong it was, and nudging its weights in the right direction. It does this thousands of times with thousands of examples. The learning rate controls how aggressive each nudge is. Over time, the weights settle into values that make the perceptron get the right answer most of the time. This feedback loop — predict, measure error, update, repeat — is the foundation of ALL machine learning training.

Classification of ML Methods

Machine learning methods fall into three main categories. The key question that determines which one you use is: "Do I have labeled data?"

1. Supervised Learning

You have a dataset where every example comes with a correct answer (label). The algorithm learns a mapping from inputs to outputs.

Supervised Learning

Training on labeled data (input-output pairs) to learn a function that can predict outputs for new, unseen inputs. The "supervision" comes from the labels — they tell the model whether its predictions are right or wrong.

Supervised learning splits into two sub-types:

Type Output Type Goal Examples
Classification Discrete categories (classes) Which category does this belong to? Spam vs not spam, cat vs dog image, pass vs fail
Regression Continuous number What is the predicted value? House price, temperature, stock price, salary
Analogy: Studying with an Answer Key

Supervised learning is like studying with an answer key. You do practice questions (training data), check your answers against the key (labels), and learn from your mistakes. Eventually, you can answer new questions correctly even without the key.

Classification = answering multiple choice (the answer is one of A, B, C, D). Regression = answering open-ended math problems (the answer is a number).

2. Unsupervised Learning

You have data but no labels. The algorithm must find structure in the data on its own.

Unsupervised Learning

Finding hidden patterns or groupings in unlabeled data. Nobody tells the model what the right answer is — it discovers structure by itself.

Common unsupervised tasks:

  • Clustering — Group similar data points together. Example: K-Means groups customers into segments based on purchasing behavior.
  • Dimensionality Reduction — Compress many features into fewer while preserving important information. Example: PCA (Principal Component Analysis) reduces 100 features to 10 for visualization.
Analogy: Sorting a Pile of Mixed Coins

Imagine someone dumps a bag of mixed coins on a table. Nobody tells you which are Malaysian, US, or Euro coins. You just start grouping them by similarity — size, color, markings. That is clustering. You found the groups without anyone labeling them.

Mapping: Coins = data points. Similarity features (size, color) = input features. Groups you form = clusters. Nobody gave you labels = unsupervised.

3. Reinforcement Learning

An agent learns by interacting with an environment. It takes actions and receives rewards (or penalties). The goal is to learn a strategy (policy) that maximizes total reward over time.

Analogy: Training a Dog

You tell the dog to sit. If it sits, you give it a treat (reward). If it barks instead, nothing (or a gentle "no" = penalty). Over time, the dog learns which actions lead to treats. It does not understand language — it learns from trial, error, and feedback.

Mapping: Dog = agent. Room/commands = environment. Sit/bark/stay = actions. Treat = positive reward. "No" = negative reward. Training over many sessions = learning over many episodes.

When to Go Deep: Feature Engineering Bottleneck

In traditional ML (like linear regression or decision trees), a human expert must select and create features from raw data. For example, to classify images, you might need to manually extract edge detectors, color histograms, and texture features. This is called feature engineering, and it is often the hardest and most time-consuming part.

Key Takeaway

Deep learning shines when feature engineering is a bottleneck. Instead of hand-crafting features, you feed raw data (pixels, text, audio) directly into a deep neural network, and it learns the features itself. This is why DL dominates in image recognition, natural language processing, and speech — domains where raw data is high-dimensional and features are hard to define manually.

Plain-English Recap

Supervised learning = learning with labeled examples (answer key provided). It splits into classification (which category?) and regression (what number?). Unsupervised learning = finding hidden patterns in data without labels (like grouping similar customers). Reinforcement learning = learning by trial and error with rewards (like training a pet). Deep learning enters the picture when feature engineering becomes too difficult for humans — the network learns its own features from raw data.

Flashcards

Machine Learning
A subset of AI where systems learn patterns from data to make predictions or decisions, rather than following explicitly programmed rules.
Perceptron
The simplest neural network unit: takes inputs, multiplies by weights, adds a bias, and applies an activation function. It is the building block of all neural networks.
Activation Function
A mathematical function applied to the weighted sum of inputs in a neuron to introduce non-linearity, enabling the network to learn complex patterns. Examples: step, sigmoid, ReLU.
Sigmoid Function
σ(z) = 1/(1+e-z). Squashes any real number into the range (0, 1). Used as an activation function to produce probability-like outputs. It is differentiable, enabling gradient-based learning.
Bias
An additional learnable parameter in a neuron, added to the weighted sum of inputs. It shifts the activation threshold, allowing the model to fit data even when all inputs are zero.
Weights
Learnable parameters that determine how much influence each input has on a neuron's output. During training, weights are adjusted to minimize prediction errors. High weight = high importance.
Supervised Learning
Training a model on labeled data (input-output pairs) to learn a mapping function. The model learns from its mistakes using the provided labels. Subtypes: classification and regression.
Unsupervised Learning
Finding hidden patterns or groupings in data without labels. The model discovers structure on its own. Common methods: K-Means clustering, PCA for dimensionality reduction.
Overfitting
When a model learns the training data too well, including its noise and quirks, and fails to generalize to new, unseen data. It memorizes instead of learning the underlying pattern.
Underfitting
When a model is too simple to capture the underlying pattern in the data. It performs poorly on both training data and new data. Often caused by insufficient features or an overly simple model.

B. z = -1
C. z = 1
D. z = 5
z = (0.5 × 4) + (1.0 × -2) + (-1) = 2 + (-2) + (-1) = -1. Each input is multiplied by its weight, then all values (including bias) are summed.

B. PCA for dimensionality reduction
C. Predicting house prices from square footage
D. An agent learning to play chess through rewards
Predicting house prices is regression (a supervised learning task) because you have labeled training data: house features paired with known prices. K-Means and PCA are unsupervised (no labels). Chess via rewards is reinforcement learning.

The three main categories of ML are: Supervised (learn from labeled data), Unsupervised (find patterns in unlabeled data), and Reinforcement (learn through trial and error with rewards). Classification/regression are subtypes of supervised learning, and the others are concepts within ML, not the main categories.

Scenario Challenge

Scenario: Choosing the Right ML Approach ▼

Scenario: You work at a hospital. A doctor asks you to help with two different problems:

  1. Problem A: Given a patient's medical history (age, blood pressure, cholesterol, BMI, etc.), predict whether they are at risk of diabetes (yes/no). You have 10,000 past patient records with known diabetes diagnoses.
  2. Problem B: You have 50,000 anonymous patient records with no labels. The doctor wants to know if there are natural groupings of patients that might reveal subtypes of a disease nobody has categorized yet.

Questions to Answer:

  1. Which ML category (supervised, unsupervised, reinforcement) would you use for Problem A? Why?
  2. Would Problem A be classification or regression? Explain.
  3. Which ML category for Problem B? Why?
  4. What specific algorithm might you use for Problem B?
  5. The doctor asks: "Can you use the same model for both problems?" What do you say?

1. Problem A → Supervised Learning. You have labeled data: 10,000 records where each patient has known features AND a known diagnosis (yes/no). The model learns from these labels.

2. Classification. The output is a discrete category: "diabetes" or "no diabetes." This is binary classification (two classes). If instead the doctor asked "what is the probability score?" it could still be framed as classification with probability output.

3. Problem B → Unsupervised Learning. You have no labels — the doctor does not even know what the groupings are. The algorithm must discover hidden structure on its own.

4. K-Means Clustering would be a natural choice. You specify K (the number of clusters, e.g., 3 or 5), and the algorithm groups patients based on feature similarity. You could then have a medical expert examine each cluster to see if they correspond to disease subtypes.

5. No, you cannot use the same model. Problem A requires labels (supervised) while Problem B has no labels (unsupervised). They are fundamentally different learning paradigms. You could, however, use the clusters from Problem B as features for a supervised model later — a technique called feature engineering.
