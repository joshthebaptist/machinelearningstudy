1. Why Sequences Matter

Analogy: Word Order Changes Everything

Consider these two sentences:

"The dog bit the man."

"The man bit the dog."

They contain exactly the same words, yet the meaning is completely opposite. What changed? Only the order. This proves that order carries meaning — and models must respect that.

A sequence is any ordered collection of elements where position matters. Unlike sets or bags-of-features, the same elements arranged differently yield different meaning.

Sequences appear everywhere:

  • Text: "I love this movie" vs "I don't love this movie" — a single word reversal flips sentiment
  • Speech: Sound waves unfold in time; misplacing a phoneme changes the word
  • Time series: Stock prices, sensor readings, ECG signals — yesterday's value contextualizes today's
  • Video: Frames in order show motion; shuffled frames are meaningless noise
  • DNA: A gene's function depends on surrounding nucleotides

Key Idea

The meaning of an element depends on what came before it. Any model that processes sequences must maintain some notion of history. This is the fundamental motivation for Recurrent Neural Networks.

2. Why Feed-Forward Networks Fail on Sequences

Standard feed-forward networks have two critical limitations when facing sequential data:

Problem 1: Fixed Input Size

A feed-forward network is built with a fixed number of input neurons. If you design it for 10 words, you cannot feed it 11 words. If you design it for 100 pixels, you cannot feed it 200. Sequences have variable lengths — a sentence can be 3 words or 30 words — and feed-forward nets simply cannot adapt.

Problem 2: No Memory

Even if you padded every input to a fixed length, the network processes all inputs in a single pass. It has no mechanism to remember what it saw at step 1 when it reaches step 5. Each input is treated independently within the forward pass.

Analogy: Reading a Sentence

Imagine reading a novel where someone shows you one word at a time on flashcards — but after each flashcard, you forget everything you've read so far. By the time you reach the period at the end of a sentence, you have no idea what the sentence was about. That is what a feed-forward network does. A human reader, by contrast, holds a running understanding of the sentence, updating their interpretation with every new word.

Definition: Recurrence

Recurrence is the architectural principle of feeding the network's own output (or hidden state) back into itself at the next time step. This creates a feedback loop that allows information to persist across steps — giving the network a form of memory.

3. The Recurrent Cell

A recurrent neural network (RNN) is built from a single recurrent cell that processes one element at a time while maintaining a hidden state — a running summary of everything it has seen so far.

Definition: Hidden State

The hidden state (denoted ht) is a vector that the RNN cell updates at each time step. It is the network's "memory" — a compressed representation of all previous inputs. At time step t, the cell reads input xt and the previous hidden state ht-1, then produces a new hidden state ht.

How it works:

  1. Read one element: The cell receives input xt (e.g., one word's embedding at time step t).
  2. Combine with memory: It merges xt with the previous hidden state ht-1 using learned weights.
  3. Produce new memory: The result is passed through a non-linearity (typically tanh) to produce ht.
  4. Feed back: ht is passed to the next time step as ht-1 for the next input.

Analogy: The Note-Taker

Imagine you are in a lecture and taking notes. At each moment, you read a new sentence from the speaker (xt), look at your running notes so far (ht-1), and then update your notes to include the new information (ht). Your notes are a compressed summary that carries forward — and you use the same pen and paper (the same weights) every time.

Key Insight: Weight Sharing

The same weights are reused at every time step. Whether the sequence has 5 elements or 500, the cell uses identical parameters. This means the network learns a general rule for processing transitions, not a rule for each position. It also means the model size is independent of sequence length.

Unrolling through time: Although we draw the RNN cell with a loop, it is often easier to reason about by "unrolling" the computation across time steps. This produces a chain of identical cells, each connected to the next through the hidden state vector.

x₁ → [Cell] → h₁ → [Cell] → h₂ → [Cell] → h₃ → ... → hₜ

Each [Cell] box uses the exact same weight matrices Wxh and Whh.

4. Sequence Task Shapes

Not all sequence problems have the same input/output structure. We classify them by how inputs and outputs relate:

Many-to-One

Input: A sequence of many elements. Output: A single value.

Examples:

  • Sentiment classification: Feed in a sequence of words → output a single sentiment score (positive/negative).
  • Risk scoring: Feed in a patient's sequence of vital signs over 24 hours → output a single risk score.

The final hidden state hT serves as the summary and is fed to a classifier.

One-to-Many

Input: A single value. Output: A sequence of many elements.

Examples:

  • Image captioning: Feed in a single image feature vector → output a sequence of words describing the image.
  • Music generation: Feed in a starting note or genre tag → output a sequence of notes.

Many-to-Many (Equal Length)

Input: A sequence. Output: A sequence of the same length, where each output corresponds to one input.

Examples:

  • POS tagging: Each word in a sentence gets a part-of-speech tag.
  • Named entity recognition: Each word is classified as person, location, organization, or other.
  • Pixel-wise segmentation: Each pixel in an image gets a class label.

The RNN outputs at every time step; each output depends on both past and (in bidirectional models) future context.

Many-to-Many (Unequal Length) — Encoder-Decoder

Input: A sequence of length M. Output: A sequence of length N, where M ≠ N.

Examples:

  • Machine translation: Translate an English sentence (M words) into a French sentence (N words, usually different count).
  • Dialogue systems: A question of varying length generates an answer of varying length.

The encoder reads the input sequence and compresses it into a context vector. The decoder generates the output sequence one element at a time, conditioned on that context.

Rule of Thumb

When facing a new sequential task, first determine the shape: How many inputs? How many outputs? Are they the same length? The shape determines your architecture.

5. RNN Forward Pass — The Math

A vanilla RNN cell performs two computations at each time step t:

Hidden state update:
ht = tanh(Wxh · xt + Whh · ht-1 + bh)
Output (optional):
ŷt = Why · ht + by

Where:

  • xt — input vector at time step t
  • ht — hidden state (memory) at time step t
  • ht-1 — hidden state from the previous time step
  • Wxh — weight matrix connecting input to hidden state
  • Whh — weight matrix connecting previous hidden state to current hidden state
  • bh — bias vector for the hidden state
  • Why — weight matrix connecting hidden state to output
  • by — bias vector for the output
  • tanh — hyperbolic tangent, squashes values to (−1, +1)
Initial condition:
h0 = 0 (zero vector)

Intuition: A Blending Function

The hidden state update is a blend of two things:

  1. What I see now: Wxh · xt — the current input, projected into hidden space.
  2. What I remembered: Whh · ht-1 — the running summary from all past inputs.

These are added together (with bias), then squashed through tanh to keep values bounded. The result is a new memory that fuses past and present. The tanh ensures values stay in (−1, +1), preventing the hidden state from growing unboundedly.

Summary

At every step, the RNN asks: "Given what I see now and what I remember, what should I remember next?" The answer is a new hidden state — a compressed, evolving summary of the entire sequence processed so far.

6. RNN Worked Example (Hand Calculation)

Let us compute the forward pass of a simple scalar RNN step by step.

Given Parameters

  • Wxh = 0.5
  • Whh = 0.8
  • bh = 0.0
  • Why = 1.0
  • by = 0.0

Inputs: x₁ = 1.0, x₂ = 0.5

Initial hidden state: h₀ = 0.0

Step 1: Time step t = 1

Input: x₁ = 1.0, Previous hidden state: h₀ = 0.0

Compute the linear combination:

Wxh · x₁ + Whh · h₀ + bh

= 0.5 × 1.0 + 0.8 × 0.0 + 0.0

= 0.5 + 0.0 + 0.0

= 0.5

Apply tanh:

h₁ = tanh(0.5)

h₁ = 0.4621

(Recall: tanh(0.5) = (e0.5 − e−0.5) / (e0.5 + e−0.5) = (1.6487 − 0.6065) / (1.6487 + 0.6065) = 1.0422 / 2.2552 = 0.4621)

Step 2: Time step t = 2

Input: x₂ = 0.5, Previous hidden state: h₁ = 0.4621

Compute the linear combination:

Wxh · x₂ + Whh · h₁ + bh

= 0.5 × 0.5 + 0.8 × 0.4621 + 0.0

= 0.25 + 0.3697

= 0.6197

Apply tanh:

h₂ = tanh(0.6197)

h₂ = 0.5509

Step 3: Output at t = 2

Compute output:

ŷ₂ = Why · h₂ + by

= 1.0 × 0.5509 + 0.0

ŷ₂ = 0.5509

Why This Matters

Notice that h₂ depends on h₁, and h₁ depended on x₁. Even though the output is at time step 2, the information from x₁ = 1.0 is encoded inside h₂ and therefore influences the output. The memory is carried forward through the hidden state chain.

Complete Trace Table

Step Input xt ht-1 Linear ht Output
t=0 — 0.0000 — 0.0000 —
t=1 1.0 0.0000 0.5000 0.4621 —
t=2 0.5 0.4621 0.6197 0.5509 0.5509

7. Vanishing Gradients in RNNs

During training, RNNs are optimized using Backpropagation Through Time (BPTT) — an extension of standard backpropagation that unrolls the RNN across all time steps and applies the chain rule through the unrolled graph.

Definition: Backpropagation Through Time (BPTT)

BPTT is the algorithm used to compute gradients in recurrent networks. The RNN is "unrolled" into a deep feed-forward network (one layer per time step), and standard backpropagation is applied. Gradients flow backward through the hidden state chain from step T back to step 1.

The gradient of the loss at step t with respect to an earlier hidden state hk (where k < t) requires multiplying the Jacobians of each intermediate step:

∂ht/∂hk = ∏j=k+1t ∂hj/∂hj-1 = ∏j=k+1t diag(1 − hj²) · Whh

The Vanishing Gradient Problem

Each Jacobian term diag(1 − hj²) · Whh is a matrix. Its eigenvalues determine whether the product grows or shrinks as more terms are multiplied:

  • Most eigenvalues < 1: Multiplying many such matrices causes the gradient to shrink exponentially toward zero. This is vanishing.
  • Some eigenvalues > 1: The gradient can grow exponentially. This is exploding.

Watch What Happens Over 20 Steps

If the typical scaling factor is λ:

λ λ10 λ20 Effect
0.9 0.349 0.122 Gradient reduced to ~12%
0.7 0.028 0.0008 Essentially zero
0.5 0.001 ≈ 0 Completely vanished

Analogy: The Telephone Game

Imagine a game of "telephone" where a message passes through 20 people. If each person retains 90% of the message, by the time it reaches person 20, only about 12% of the original meaning survives. If each person retains 50%, the message is completely garbled after a few steps. This is exactly what happens to gradients: the signal degrades exponentially as it travels backward through time.

Practical Consequence

A vanilla RNN can effectively remember only about 5 to 10 time steps. Dependencies beyond this range are nearly impossible to learn because the gradient signal has vanished to near-zero. This is why we need LSTM (and GRU) architectures.

8. LSTM — The Protected Memory Lane

The Long Short-Term Memory (LSTM) network was specifically designed to solve the vanishing gradient problem. Its core innovation is the cell state — a separate memory channel that runs through time with minimal interference.

Definition: Cell State (Ct)

The cell state is an additional vector maintained by the LSTM alongside the hidden state. It acts as a conveyor belt that carries information across many time steps with almost no transformation. Information is added to or removed from the cell state through learned gates, but the belt itself allows gradients to flow nearly undiminished.

Analogy: The Conveyor Belt

Imagine a factory conveyor belt running through a series of workstations. Items on the belt travel long distances without being altered — the belt itself is just transportation. Workstations can add new items to the belt, remove existing items, or peek at what's on the belt without modifying it. This is how the cell state works: it is a highway for information that bypasses the repeated nonlinear transformations that cause gradients to vanish in vanilla RNNs.

Why LSTMs Work

In a vanilla RNN, the hidden state is updated as: ht = tanh(...). Every step applies a nonlinear squashing function, which repeatedly multiplies gradients by numbers less than 1. In an LSTM, the cell state is updated additively:

Ct = ft ⊙ Ct-1 + it ⊙ Ĉt

The gradient of Ct with respect to Ct-1 is simply the forget gate ft — a vector of values between 0 and 1, but not further compressed by tanh. If the forget gate is close to 1, the gradient passes through almost perfectly. This is the key to learning long-range dependencies.

9. LSTM Gate Equations (Full)

An LSTM cell at time step t receives three inputs: the current input xt, the previous hidden state ht-1, and the previous cell state Ct-1. It computes four quantities using learned weight matrices and biases.

First, the previous hidden state and current input are concatenated: [ht-1, xt]. All four computations use this same concatenation as input.

Forget Gate:
ft = σ(Wf · [ht-1, xt] + bf)
Decides how much of the old cell state Ct-1 to keep. Values near 1 = keep; near 0 = erase.
Input Gate:
it = σ(Wi · [ht-1, xt] + bi)
Decides how much new information to admit into the cell state.
Candidate (Proposed New Content):
Ĉt = tanh(WC · [ht-1, xt] + bC)
Creates a vector of candidate values that could be added to the cell state.
Cell State Update:
Ct = ft ⊙ Ct-1 + it ⊙ Ĉt
Erases old memory (forget gate) and adds new content (input gate × candidate). The ⊙ denotes element-wise multiplication.
Output Gate:
ot = σ(Wo · [ht-1, xt] + bo)
Decides how much of the cell state to expose as the hidden state output.
Hidden State Output:
ht = ot ⊙ tanh(Ct)
The visible output. The cell state is squashed through tanh (bounded to −1,+1) and filtered by the output gate.

Analogy: Reading a Book with a Notepad

  • Forget gate = Crossing out old notes that are no longer relevant ("I don't need the character list from chapter 1 anymore").
  • Input gate = Deciding which new details are worth writing down ("This character's new motivation is important").
  • Candidate = The actual new note you write ("Elara now wants to find the crystal").
  • Cell state update = Updating your notepad — erase old stuff, write new stuff.
  • Output gate = Deciding what to mention when someone asks "what happened so far?" — you don't recite every note, just the relevant parts.

Important: Four Weight Matrices

An LSTM has four sets of weights and biases (Wf, Wi, WC, Wo and their biases), compared to the vanilla RNN's two (Wxh, Whh). This means an LSTM has roughly 4× more parameters than a vanilla RNN of the same hidden size. The extra capacity is what enables it to learn complex gating behaviors.

10. LSTM Worked Example (Hand Calculation)

Let us walk through a single LSTM time step with concrete numbers.

Given Values

  • Ct-1 = 0.60 (previous cell state)
  • ht-1 = 0.10 (previous hidden state)
  • xt = 0.50 (current input)

Pre-activations (values after linear transformation, before activation):

  • zf = 1.0 (for forget gate)
  • zi = 0.5 (for input gate)
  • zC = 0.8 (for candidate)
  • zo = 0.2 (for output gate)

Step 1: Compute All Gates

Forget gate:

ft = σ(1.0) = 1 / (1 + e−1.0) = 1 / (1 + 0.3679) = 1 / 1.3679 = 0.731

→ Keep 73.1% of old cell state

Input gate:

it = σ(0.5) = 1 / (1 + e−0.5) = 1 / (1 + 0.6065) = 1 / 1.6065 = 0.622

→ Admit 62.2% of new candidate content

Candidate:

Ĉt = tanh(0.8) = 0.664

→ Proposed new content is 0.664

Output gate:

ot = σ(0.2) = 1 / (1 + e−0.2) = 1 / (1 + 0.8187) = 1 / 1.8187 = 0.550

→ Expose 55.0% of the cell state

Step 2: Update Cell State

Ct = ft ⊙ Ct-1 + it ⊙ Ĉt

= 0.731 × 0.60 + 0.622 × 0.664

= 0.4386 + 0.4130

Ct = 0.8516

Old memory (0.4386 retained) + New content (0.4130 added) = Updated memory (0.8516)

Step 3: Compute Hidden State Output

ht = ot ⊙ tanh(Ct)

= 0.550 × tanh(0.8516)

= 0.550 × 0.6917

ht = 0.3804

Read-Out: What Happened

The belt kept 73% of the old memory (0.4386 out of 0.60) and added new content worth 0.4130. The cell state grew from 0.60 to 0.8516. The output gate then exposed only 55% of the squashed cell state to the visible hidden output. Notice that the cell state (0.8516) is larger than the old cell state (0.60) — the LSTM learned to accumulate information here.

Complete LSTM Trace Table

Quantity Value Interpretation
ft (forget gate) 0.731 Keep 73.1% of old memory
it (input gate) 0.622 Admit 62.2% of new info
Ĉt (candidate) 0.664 Proposed new content
Ct (new cell state) 0.8516 Updated memory (was 0.60)
ot (output gate) 0.550 Expose 55.0% of cell
ht (hidden output) 0.3804 Visible output

11. RNN vs LSTM vs GRU — Comparison

There are three main recurrent architectures. Here is how they compare:

Feature Vanilla RNN LSTM GRU
Gates None 3 (forget, input, output) 2 (reset, update)
Memory states 1 (ht) 2 (ht, Ct) 1 (ht)
Long-range dependencies Weak (~5-10 steps) Strong (100+ steps) Strong (100+ steps)
Number of parameters Lowest Highest (~4× vanilla) Medium (~3× vanilla)
Training cost Lowest Highest Medium
Inference speed Fastest Slowest Medium
Vanishing gradients? Yes (severe) Mitigated Mitigated
Introduced ~1990 1997 (Hochreiter & Schmidhuber) 2014 (Cho et al.)

Analogy: Note-Taking Strategies

  • Vanilla RNN: Like writing a single running paragraph. As it gets longer, early details fade and get overwritten. Good for short notes.
  • LSTM: Like having a detailed notebook with separate pages. You can earmark pages to keep, tear out pages you don't need, and add new pages. Nothing gets overwritten accidentally. Best for long, complex projects.
  • GRU: Like a simplified notebook — fewer pages, but still organized. Faster to write in, easier to manage, and almost as effective as the full notebook for most tasks.

Rule of Thumb

Start with GRU or LSTM for any task involving long-range dependencies. Use GRU first (simpler, fewer parameters) and only switch to LSTM if you need the extra expressiveness. Use vanilla RNNs only for very short sequences or as a pedagogical starting point.

Modern Note

In practice today, Transformers have largely replaced RNNs and LSTMs for most NLP tasks, as they handle long-range dependencies even more effectively through self-attention. However, RNNs and LSTMs remain important for streaming/real-time applications, edge devices with limited memory, and understanding the foundations of sequence modeling.

Flashcards

RNN
A neural network with feedback loops that allow it to maintain a hidden state across time steps. It processes sequences one element at a time, updating its memory at each step using shared weights.
LSTM
A gated recurrent architecture with a dedicated cell state (conveyor belt) that allows gradients to flow across many time steps without vanishing. Uses forget, input, and output gates to control information flow.
Hidden State
The running summary vector ht that encodes all information the network has processed so far. It is updated at each time step and passed to the next step as the network's memory.
Vanishing Gradient
During backpropagation through time, gradients are multiplied by matrices with eigenvalues < 1 at each step, causing them to shrink exponentially. This makes it impossible for vanilla RNNs to learn long-range dependencies.
BPTT
An extension of backpropagation for recurrent networks. The RNN is unrolled into a deep feed-forward network (one layer per time step), and the chain rule is applied through the entire unrolled graph to compute gradients.
Forget Gate
It decides how much of the old cell state to keep. Computed as ft = σ(Wf · [ht-1, xt] + bf). Values near 1 retain old memory; values near 0 erase it.
Input Gate
It decides how much new information to write into the cell state. Computed as it = σ(Wi · [ht-1, xt] + bi). Works with the candidate vector Śt to determine what gets added.
Output Gate
It decides how much of the cell state to expose as the hidden state output. Computed as ot = σ(Wo · [ht-1, xt] + bo). The cell state is first squashed through tanh, then filtered by the output gate.
Cell State
A separate memory vector Ct that runs through time like a conveyor belt. Updated additively (not through repeated tanh), which allows gradients to flow across many steps with minimal decay.
Sequence Task
Many-to-one (sentiment analysis), One-to-many (image captioning), Many-to-many equal (POS tagging), Many-to-many unequal (machine translation via encoder-decoder).
GRU
A simplified recurrent architecture with only two gates (reset and update) and a single state (ht). Fewer parameters than LSTM but comparably effective for many tasks. Introduced by Cho et al. in 2014.
tanh
tanh squashes values to the range (−1, +1), keeping hidden states and cell states bounded. Unlike sigmoid (0 to 1), tanh is centered at zero, which helps with learning dynamics. It introduces non-linearity needed for the network to learn complex patterns.

Scenario Question

You are building a model to predict tomorrow's stock price based on the last 30 days of daily closing prices. What type of sequence task shape is this, and which architecture would you choose?

Click to reveal answer

Task shape: Many-to-one. You have 30 input values (many) and produce 1 output (single price).

Architecture choice: Start with a GRU or LSTM. A vanilla RNN would struggle because stock prices from 25+ days ago may still be relevant (e.g., trend information), and the vanishing gradient would prevent the model from learning those long-range patterns. An LSTM's cell state can preserve trend information across all 30 days. A GRU would also work well with fewer parameters and faster training.

Why not a Transformer? For a 30-step sequence with limited training data, an LSTM/GRU is more data-efficient and less likely to overfit than a Transformer, which needs more data to learn attention patterns effectively.
