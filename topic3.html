1. Why Logistic Regression

Analogy: Imagine you're a teacher grading exams on a 0–100 scale. Linear regression would happily predict a score of 142 or -37 — both meaningless as probabilities. You need a function that squashes any real number into the [0, 1] range, just like a thermostat that can only output between "fully off" and "fully on."

Mapping:
  • Raw exam score (unbounded) → linear regression output (unbounded)
  • Percentage chance of passing (bounded 0–1) → logistic regression output (bounded 0–1)
  • Outlier student scoring 200 → outlier data point shifting the regression line

The Problem with Linear Regression for Classification

Suppose we want to predict whether a patient has a disease (1) or not (0) based on a biomarker. We might try fitting a linear regression:

ŷ = w₁x + b

This has two fatal flaws:

Flaw 1: Unbounded Output. If x = 10, w₁ = 2, b = -3, then ŷ = 17. But a probability cannot exceed 1. Similarly, ŷ can go negative — probabilities cannot be negative either. The model can produce any real number, but we need outputs strictly in [0, 1].
Flaw 2: Outlier Sensitivity. A single extreme data point far from the rest pulls the entire regression line toward it, changing the decision boundary. If you add one patient with biomarker = 1000, the slope flattens dramatically and previously correct classifications become wrong.

What We Actually Want

We don't want a raw score — we want a probability:

P(y = 1 | x) = ?

This probability must satisfy:

  • 0 ≤ P(y=1|x) ≤ 1
  • If P(y=1|x) > 0.5, predict class 1
  • If P(y=1|x) < 0.5, predict class 0
  • If P(y=1|x) = 0.5, maximum uncertainty

The Decision Boundary

The decision boundary is the threshold where the model switches its prediction. For logistic regression, this is where σ(z) = 0.5, which occurs at z = 0 (i.e., wᵀx + b = 0). This boundary is a hyperplane in the feature space.

Definition — Decision Boundary: The set of points where the predicted probability equals 0.5 (or equivalently, where z = wᵀx + b = 0). Points on one side are classified as class 1; points on the other side as class 0.
Key Takeaway: Linear regression produces unbounded outputs and is sensitive to outliers. We need a model that outputs probabilities in [0, 1] and has a well-defined decision boundary. That model is logistic regression.

2. The Sigmoid Function

Analogy: Think of a water faucet. No matter how hard you turn the handle (the input z), the flow rate can never exceed the maximum capacity or go below zero. The sigmoid function is like a faucet that smoothly transitions from "fully off" (0) to "fully on" (1) as you turn the handle.

Mapping:
  • Handle position (turn left = negative, right = positive) → linear score z
  • Flow rate (0 to max) → probability σ(z) in [0, 1]
  • Fully off → σ(z) ≈ 0 (class 0)
  • Fully on → σ(z) ≈ 1 (class 1)
  • Middle position → σ(z) = 0.5 (uncertain)

Definition

Definition — Sigmoid Function:
σ(z) = 1 / (1 + e−z)
where z = wᵀx + b = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

Properties of the Sigmoid

The sigmoid function has several critical properties that make it ideal for classification:

  • Bounded: Output is always in (0, 1), regardless of how large or small z is
  • S-shaped curve: Starts near 0, rises through 0.5 at z=0, and approaches 1
  • Symmetric at origin: σ(-z) = 1 - σ(z), meaning it's symmetric about the point (0, 0.5)
  • Smooth and differentiable: No sharp corners, gradient exists everywhere
  • Monotonically increasing: Larger z always gives larger σ(z)

Full Pipeline: From Features to Class Prediction

Worked Example — Email Spam Classification:

Given features: x = [word_freq("free") = 3.2, word_freq("meeting") = 0.1, caps_ratio = 0.4]

Weights: w = [0.5, -0.3, 1.2], bias: b = -1.0

Step 1: Compute linear score z

z = wᵀx + b = (0.5)(3.2) + (-0.3)(0.1) + (1.2)(0.4) + (-1.0)
z = 1.6 + (-0.03) + 0.48 - 1.0 = 1.05

Step 2: Apply sigmoid

σ(1.05) = 1 / (1 + e-1.05) = 1 / (1 + 0.350) = 1 / 1.350 ≈ 0.741

Step 3: Interpret probability

P(spam | x) = 0.741 → Since 0.741 > 0.5, predict class 1 (spam).

Step 4: Decision

The model is 74.1% confident this email is spam. The decision boundary is crossed when z = 0 (i.e., when wᵀx + b = 0).

Key Takeaway: The sigmoid function σ(z) = 1/(1+e⁻ᶻ) maps any real number z to a value in (0, 1), making it a natural choice for representing probabilities. The full pipeline is: compute linear score → pass through sigmoid → interpret as probability → threshold at 0.5 for classification.

3. Sigmoid Derivative

Analogy: Imagine a car's speedometer. The derivative of position is speed. For the sigmoid, the derivative tells us how fast the output is changing at any point. At z = 0, the curve is steepest — the sigmoid is most sensitive to changes there. At the extremes (z ≈ ±10), the curve is nearly flat — the sigmoid barely responds to input changes.

Mapping:
  • Position of car → σ(z) (sigmoid output)
  • Speed of car → dσ/dz (rate of change of sigmoid)
  • Steepest part of road (z=0) → maximum gradient = 0.25
  • Flat highway (z≈±10) → gradient ≈ 0 (vanishing gradient problem)

The Derivative

Definition — Sigmoid Derivative:
dσ/dz = σ(z) · (1 − σ(z))

Proof (Step by Step)

Worked Example — Deriving dσ/dz:

Starting with σ(z) = 1 / (1 + e−z) = (1 + e−z)−1

Step 1: Apply chain rule

dσ/dz = −1 · (1 + e−z)−2 · (−e−z) = e−z / (1 + e−z)2

Step 2: Rewrite as product of σ(z) and (1−σ(z))

Notice that e−z / (1 + e−z)2 = [1 / (1 + e−z)] · [e−z / (1 + e−z)]

And e−z / (1 + e−z) = 1 − 1/(1+e−z) = 1 − σ(z)

Therefore: dσ/dz = σ(z) · (1 − σ(z))

Why This Matters

Why the self-referential form matters:
  • Computationally efficient: If you've already computed σ(z) during the forward pass, the derivative costs only one multiplication and one subtraction — no new exponentials needed.
  • No extra memory: You don't need to store intermediate values; σ(z) is already available.
  • Foundation for neural networks: This same pattern — the derivative expressed in terms of the forward pass output — generalizes to many activation functions and is the backbone of backpropagation.

Maximum Derivative Value

The maximum value of σ(z)·(1−σ(z)) occurs at z = 0 where σ(0) = 0.5:

max dσ/dz = 0.5 × (1 − 0.5) = 0.25

This means the sigmoid's gradient is at most 0.25. In deep networks with many sigmoid layers, gradients shrink exponentially — the vanishing gradient problem. This is why modern neural networks prefer ReLU activation.

Key Takeaway: dσ/dz = σ(z)(1−σ(z)). This elegant self-referential form means the derivative can be computed cheaply from the forward pass output. Maximum gradient is 0.25 at z=0, which causes vanishing gradients in deep networks.

4. Odds and Logit

Analogy: Suppose you're told "the chance of rain is 75%." That's a probability. But bookmakers talk differently — they say "3 to 1 odds." Odds = 3 means for every 1 time it doesn't rain, it rains 3 times. The logit function is the natural logarithm of these odds, and logistic regression models the logit as a linear function of the features.

Mapping:
  • Probability P(rain) = 0.75 → Odds = 0.75/0.25 = 3 → Log-odds = ln(3) ≈ 1.10
  • Probability P(rain) = 0.01 → Odds = 0.01/0.99 ≈ 0.0101 → Log-odds ≈ -4.60
  • Linear model predicts log-odds → sigmoid converts back to probability

Odds

Definition — Odds:
Odds = P(y=1|x) / (1 − P(y=1|x))

Odds range from 0 to ∞. Odds = 1 means equal chance. Odds > 1 means more likely to occur. Odds < 1 means less likely.

Logit Function

Definition — Logit (Log-Odds):
logit(P) = ln(P / (1 − P)) = z = wᵀx + b

The logit maps probabilities from [0, 1] to (−∞, +∞), making them suitable for linear modeling.

Logistic Regression is Linear in Log-Odds

This is the key insight. The model says:

ln(P / (1−P)) = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

Each weight wᵢ has a clear interpretation:

  • A unit increase in xᵢ changes the log-odds by wᵢ
  • Equivalently, a unit increase in xᵢ multiplies the odds by ewᵢ
  • ewᵢ is called the odds ratio for feature xᵢ
Worked Example — Hospital Readmission:

Model: ln(P(readmit)/(1−P(readmit))) = 0.8 × age + 0.3 × severity − 1.2

Interpreting w₁ = 0.8 (age):

A one-unit increase in age (say, 1 year) changes log-odds by 0.8.

Odds ratio = e0.8 = 2.2255

This means each additional year of age multiplies the odds of readmission by 2.23. In other words, a 61-year-old has 2.23× the odds of readmission compared to a 60-year-old, holding severity constant.

Interpreting w₂ = 0.3 (severity):

Odds ratio = e0.3 = 1.3499

Each one-unit increase in severity score multiplies readmission odds by 1.35 (a 35% increase).

Interpreting b = −1.2 (bias):

At the baseline (age=0, severity=0):

P(readmit) = σ(−1.2) = 1/(1+e1.2) = 1/(1+3.32) ≈ 0.231

A hypothetical patient with age=0 and severity=0 has a 23.1% baseline readmission probability.

Concrete prediction for age=50, severity=3:

z = 0.8(50) + 0.3(3) − 1.2 = 40 + 0.9 − 1.2 = 39.7
P = σ(39.7) ≈ 1.0 (virtually certain readmission)
Common Mistake: Odds ratios are multiplicative, not additive. An odds ratio of 2.23 does NOT mean the probability increases by 123%. It means the ratio P/(1−P) is multiplied by 2.23. The actual probability change depends on the baseline.
Key Takeaway: Logistic regression models log-odds as a linear function. Each weight wᵢ tells you how the log-odds change per unit increase in xᵢ. The odds ratio ewᵢ gives the multiplicative effect on the odds. This interpretation makes logistic regression highly valuable in fields like medicine and social science.

5. Why Not MSE for Classification?

Analogy: Imagine trying to find the lowest point in a valley. If the valley has only one bottom (convex), you'll always reach it by walking downhill. But if the valley has multiple pits (non-convex), you might walk into a shallow pit and think you've reached the bottom — when the true bottom is elsewhere. Using MSE with sigmoid creates a landscape full of these deceptive pits.

Mapping:
  • Valley landscape → loss surface as function of weights
  • One bottom (convex) → BCE loss surface
  • Multiple pits (non-convex) → MSE + sigmoid loss surface
  • Walking downhill → gradient descent
  • Getting stuck in a shallow pit → local minimum

MSE with Sigmoid: The Problem

If we define the loss as MSE:

L_MSE = (1/N) Σ (y − σ(wᵀx + b))²

The combination of the squared error with the sigmoid produces a non-convex loss surface. This means:

  • The loss surface has multiple local minima
  • Gradient descent can get stuck in a suboptimal local minimum
  • Small changes in initial weights can lead to very different final models
  • No guarantee of finding the globally best solution
Visual intuition: Plot the MSE loss as a function of z for a single sample with y=1. You'll see the curve has two "bumps" — one near z=−5 and another near z=+5 — with a plateau in between. Gradient descent starting from z=0 might descend into the wrong bump or get stuck on the plateau.

BCE: The Solution

Why Binary Cross-Entropy is convex:
  • BCE loss: L = −[y·log(σ(z)) + (1−y)·log(1−σ(z))]
  • This loss is convex with respect to z (and thus with respect to w and b)
  • A convex function has exactly one global minimum and no local minima
  • Gradient descent is guaranteed to converge to the global minimum (with appropriate learning rate)
Worked Example — Comparing Loss Surfaces:

For a single sample with y=1, compare MSE and BCE at different predicted probabilities:

Predicted σ(z)MSE Loss = (1−ŷ)²BCE Loss = −log(ŷ)

BCE penalizes confident wrong predictions much more aggressively. When ŷ=0.1 for a positive sample, BCE=2.30 but MSE=0.81. This strong penalty drives the model to correct confident mistakes quickly.

Key Takeaway: MSE + sigmoid creates a non-convex loss surface with local minima, making gradient descent unreliable. BCE loss is convex, guaranteeing a single global minimum. This is why we use BCE (not MSE) for binary classification.

6. Binary Cross-Entropy (BCE) Derivation

Analogy — Maximum Likelihood Estimation: Imagine you're a detective. You see footprints (data) and want to find the most likely culprit (model parameters). MLE asks: "Which parameter values make the observed data most probable?" You examine every suspect and pick the one whose shoe size, walking pattern, etc. best explain the footprints you found.

Mapping:
  • Footprints → observed data (x, y)
  • Suspect characteristics → model parameters (w, b)
  • "How likely are these footprints given this suspect?" → likelihood function L(w, b | data)
  • "Most likely suspect" → argmax L(w, b | data) → MLE estimate
  • Converting product of probabilities to sum via log → log-likelihood

Step 0: Maximum Likelihood Estimation (MLE) Intuition

We want to find w and b that maximize the probability of observing our training data. For each data point (xᵢ, yᵢ), the model's predicted probability is:

P(y = yᵢ | xᵢ) = ŷᵢyᵢ · (1 − ŷᵢ)(1−yᵢ)

where ŷᵢ = σ(wᵀxᵢ + b).

Why this formula? Because:

  • If yᵢ = 1: P = ŷᵢ¹ · (1−ŷᵢ)⁰ = ŷᵢ ✓
  • If yᵢ = 0: P = ŷᵢ⁰ · (1−ŷᵢ)¹ = 1−ŷᵢ ✓

Step 1: The Log Trick (Product → Sum)

The likelihood over all N independent samples is the product:

L(w, b) = ∏ᵢ ŷᵢyᵢ · (1 − ŷᵢ)(1−yᵢ)

Taking the log converts the product to a sum (easier to compute, same maximum):

log L(w, b) = Σᵢ [yᵢ · log(ŷᵢ) + (1 − yᵢ) · log(1 − ŷᵢ)]
Why the log trick works: log is a monotonically increasing function, so the value of w, b that maximizes log L is the same as the one that maximizes L. But log L is a sum instead of a product — much easier to differentiate and less prone to numerical underflow (products of many small numbers become zero in floating-point arithmetic).

Step 2: Negate (Maximize → Minimize)

Standard optimization algorithms (gradient descent) minimize a loss function. So we negate the log-likelihood to convert from maximization to minimization:

L(w, b) = −Σᵢ [yᵢ · log(ŷᵢ) + (1 − yᵢ) · log(1 − ŷᵢ)]

Step 3: Average Over N Samples

We divide by N to get the mean loss, making it independent of dataset size:

L_BCE = −(1/N) Σᵢ₌₁ᴺ [yᵢ · log(ŷᵢ) + (1 − yᵢ) · log(1 − ŷᵢ)]
Definition — Binary Cross-Entropy Loss:
L_BCE = −(1/N) Σᵢ₌₁ᴺ [yᵢ · log(ŷᵢ) + (1 − yᵢ) · log(1 − ŷᵢ)]

where ŷᵢ = σ(wᵀxᵢ + b) is the predicted probability, yᵢ ∈ {0, 1} is the true label, and N is the number of samples.

Worked Example — Computing BCE for 3 Samples:

True labels: y = [1, 0, 1]

Predicted probabilities: ŷ = [0.9, 0.2, 0.7]

Sample 1 (y=1, ŷ=0.9):

L₁ = −[1·log(0.9) + 0·log(0.1)] = −log(0.9) = −(−0.105) = 0.105

Sample 2 (y=0, ŷ=0.2):

L₂ = −[0·log(0.2) + 1·log(0.8)] = −log(0.8) = −(−0.223) = 0.223

Sample 3 (y=1, ŷ=0.7):

L₃ = −[1·log(0.7) + 0·log(0.3)] = −log(0.7) = −(−0.357) = 0.357

Final BCE Loss:

L_BCE = (0.105 + 0.223 + 0.357) / 3 = 0.685 / 3 ≈ 0.228
Key Takeaway: BCE is derived from Maximum Likelihood Estimation. The steps are: (1) write the likelihood as a product, (2) take the log to convert product to sum, (3) negate to convert maximization to minimization, (4) average over samples. Final form: L = −(1/N)Σ[y·log(ŷ) + (1−y)·log(1−ŷ)].

7. BCE Intuition

Analogy: Think of BCE as a strict teacher grading true/false exams. If a student answers "True" and is correct with 99% confidence, they get almost no penalty (loss ≈ 0.01). But if they answer "True" with only 10% confidence — they barely committed to their answer — the penalty is enormous (loss ≈ 2.30). The teacher penalizes not just wrong answers, but lack of confidence in correct answers.

Mapping:
  • Student answers True confidently (ŷ=0.99, y=1) → low loss (0.01) → teacher is pleased
  • Student answers True hesitantly (ŷ=0.51, y=1) → moderate loss (0.67) → teacher wants more commitment
  • Student answers True but is wrong (ŷ=0.90, y=0) → high loss (2.30) → teacher is harsh
  • Student is very wrong and very confident → penalty approaches infinity

Case 1: True Label y = 1

When the true label is 1, the BCE loss simplifies to:

L = −log(ŷ)

0.900.105Correct and fairly confident → small penalty

As ŷ → 0 (confidently predicting class 0 when true label is 1), loss → ∞. The model is severely punished for being confidently wrong.

Case 2: True Label y = 0

When the true label is 0, the BCE loss simplifies to:

L = −log(1 − ŷ)
Predicted ŷ1−ŷLoss −log(1−ŷ)Behavior

Confidence-Penalizing Property: BCE acts as a confidence penalty. It doesn't just care about being right or wrong — it cares about how sure the model is. Being wrong with high confidence is the worst case. This pushes the model to not only be correct, but to be well-calibrated in its confidence.
Worked Example — Two Models Compared:

True label: y = 1

Model A predicts: ŷ = 0.6 (correct but uncertain)

Model B predicts: ŷ = 0.95 (correct and confident)

L_A = −log(0.6) = 0.511
L_B = −log(0.95) = 0.051

Model B is preferred: both are correct, but B is 10× more confident. BCE rewards this confidence.

Now consider:

Model C predicts: ŷ = 0.4 (incorrect, predicts class 0)

Model D predicts: ŷ = 0.01 (very incorrect, very confident class 0)

L_C = −log(0.4) = 0.916
L_D = −log(0.01) = 4.605

Both are wrong, but D is punished 5× more because it was confidently wrong. This is exactly what we want — the model should learn to avoid confident mistakes.

Key Takeaway: BCE is a confidence-penalizing loss. For y=1, loss = −log(ŷ) penalizes low predicted probabilities. For y=0, loss = −log(1−ŷ) penalizes high predicted probabilities. The penalty approaches infinity for confidently wrong predictions, driving the model to be both correct and well-calibrated.

8. Gradient of BCE & Update Rule

Analogy: You're hiking downhill blindfolded, and the slope of the terrain tells you which direction to step. The gradient of the BCE loss tells you exactly how much each weight contributed to the error and in which direction to adjust it. The learning rate α is your step size — too large and you overshoot; too small and it takes forever.

Mapping:
  • Position on mountain → current weight values (w, b)
  • Altitude → loss value L_BCE
  • Slope in each direction → partial derivatives ∂L/∂wᵢ, ∂L/∂b
  • Step size → learning rate α
  • Walking downhill → gradient descent (moving opposite to gradient)

The Gradient

The key result (which you can derive using the chain rule) is beautifully simple:

Definition — BCE Gradients:
∂L/∂w = (1/N) Xᵀ(ŷ − y)
∂L/∂b = (1/N) Σᵢ (ŷᵢ − yᵢ) = mean(ŷ − y)

where ŷ is the vector of predictions and y is the vector of true labels.

Elegant result: The gradient of BCE with respect to the weights is simply the prediction error (ŷ − y) scaled by the input features. This is the same form as the gradient for linear regression with MSE — a deep mathematical coincidence that makes implementation clean.

Update Rule

w ← w − α · (1/N) Xᵀ(ŷ − y)
b ← b − α · mean(ŷ − y)

For a single sample, the per-sample update is:

wᵢ ← wᵢ − α · (ŷ − y) · xᵢ
b ← b − α · (ŷ − y)

Hand-Traced Numerical Example

Worked Example — Single Gradient Step:

Given:

  • Input: x = [2.0, 1.5]
  • True label: y = 1
  • Initial weights: w = [0.0, 0.0]
  • Initial bias: b = 0.0
  • Learning rate: α = 0.1

Step 1: Compute z = wᵀx + b

z = (0.0)(2.0) + (0.0)(1.5) + 0.0 = 0.0

Step 2: Compute ŷ = σ(z)

ŷ = σ(0.0) = 1/(1 + e⁰) = 1/(1+1) = 0.5

Step 3: Compute error = ŷ − y

error = 0.5 − 1.0 = −0.5

Step 4: Compute gradients

∂L/∂w₁ = error · x₁ = (−0.5)(2.0) = −1.0
∂L/∂w₂ = error · x₂ = (−0.5)(1.5) = −0.75
∂L/∂b = error = −0.5

Step 5: Update weights

w₁ ← 0.0 − (0.1)(−1.0) = 0.0 + 0.1 = 0.1
w₂ ← 0.0 − (0.1)(−0.75) = 0.0 + 0.075 = 0.075
b ← 0.0 − (0.1)(−0.5) = 0.0 + 0.05 = 0.05

Step 6: Verify — compute new prediction

z_new = (0.1)(2.0) + (0.075)(1.5) + 0.05 = 0.2 + 0.1125 + 0.05 = 0.3625
ŷ_new = σ(0.3625) = 1/(1 + e−0.3625) = 1/(1 + 0.696) ≈ 0.5895

Check: The prediction moved from 0.5 → 0.5895, getting closer to the true label y=1. ✓

Worked Example — Second Gradient Step:

Continuing from above with new w = [0.1, 0.075], b = 0.05:

Step 1: z = (0.1)(2.0) + (0.075)(1.5) + 0.05 = 0.3625

Step 2: ŷ = σ(0.3625) ≈ 0.5895

Step 3: error = 0.5895 − 1.0 = −0.4105

w₁ ← 0.1 − (0.1)(−0.4105)(2.0) = 0.1 + 0.0821 = 0.1821
w₂ ← 0.075 − (0.1)(−0.4105)(1.5) = 0.075 + 0.0616 = 0.1366
b ← 0.05 − (0.1)(−0.4105) = 0.05 + 0.0411 = 0.0911

New prediction: z = (0.1821)(2.0) + (0.1366)(1.5) + 0.0911 = 0.3642 + 0.2049 + 0.0911 = 0.6602

ŷ = σ(0.6602) ≈ 0.6592

Prediction improved from 0.5895 → 0.6592, continuing toward y=1. ✓

After many more steps, ŷ will approach 1.0.

Key Takeaway: The BCE gradient simplifies to (prediction error) × (input features). The update rule w ← w − α·(ŷ−y)·x is intuitive: if we underpredicted (ŷ−y < 0), increase weights on active features. The hand-traced example showed how each step pushes the prediction closer to the true label.

9. Overfitting & Regularisation

Analogy: Imagine a student memorizing every answer to practice exams (overfitting) versus understanding the underlying concepts (generalization). Regularisation is like a teacher saying "simplify your approach — don't overthink it." L2 regularisation says "keep all your notes short" (shrinks everything), while L1 says "cut out unnecessary notes entirely" (feature selection).

Mapping:
  • Memorizing practice exams → overfitting to training data
  • Performing well on new exams → generalization to test data
  • Teacher saying "simplify" → regularisation penalty
  • L2: "Keep all notes short" → shrink all weights toward 0
  • L1: "Delete unnecessary notes" → force some weights to exactly 0
  • C = 1/λ → how strict the teacher is (low C = strict, high C = lenient)

Overfitting in Logistic Regression

Definition — Overfitting: When a model learns the training data too well, including noise and outliers, resulting in high training accuracy but poor test accuracy. In logistic regression, this manifests as very large weight values that create an overly complex decision boundary.

L2 Regularisation (Ridge)

L2 adds a penalty proportional to the sum of squared weights:

L_regularised = L_BCE + λ · ||w||² = L_BCE + λ · Σᵢ wᵢ²
  • λ (lambda) controls the strength of regularisation
  • Large λ → stronger penalty → weights shrink toward 0
  • λ = 0 → no regularisation → original BCE
  • L2 shrinks all weights but rarely makes them exactly 0
  • Encourages weights to be small and evenly distributed

L1 Regularisation (Lasso)

L1 adds a penalty proportional to the sum of absolute weights:

L_regularised = L_BCE + λ · ||w||₁ = L_BCE + λ · Σᵢ |wᵢ|
  • L1 pushes many weights to exactly 0
  • This performs automatic feature selection
  • Result: a sparse model that uses only the most important features
  • Especially useful when you have many features but suspect only a few matter
Key Difference — L1 vs L2:
  • L2 (Ridge): All weights get smaller, but none become exactly 0. Good when all features are somewhat useful.
  • L1 (Lasso): Some weights become exactly 0. Good for feature selection when many features are irrelevant.
  • Elastic Net combines both: L = L_BCE + λ₁||w||₁ + λ₂||w||²

The C Parameter in scikit-learn

Definition — C Parameter:
C = 1/λ
  • Small C (e.g., 0.001): Strong regularisation → simple model → may underfit
  • Large C (e.g., 1000): Weak regularisation → complex model → may overfit
  • C = 1: Default, moderate regularisation
Worked Example — Regularisation Effect:

Suppose L_BCE = 0.35 and current weights are w = [3.0, −2.5, 0.8, −0.1] with λ = 0.1

L2 penalty:

λ||w||² = 0.1 × (3.0² + (−2.5)² + 0.8² + (−0.1)²)
= 0.1 × (9.0 + 6.25 + 0.64 + 0.01) = 0.1 × 15.9 = 1.59

Total L2 loss = 0.35 + 1.59 = 1.94

The large weights (3.0 and −2.5) contribute the most to the penalty. Gradient descent will shrink them preferentially.

L1 penalty:

λ||w||₁ = 0.1 × (|3.0| + |−2.5| + |0.8| + |−0.1|)
= 0.1 × (3.0 + 2.5 + 0.8 + 0.1) = 0.1 × 6.4 = 0.64

Total L1 loss = 0.35 + 0.64 = 0.99

L1 pushes the small weight (−0.1) toward exactly 0, effectively removing that feature.

Key Takeaway: Overfitting occurs when the model is too complex. L2 regularisation shrinks all weights; L1 regularisation can force weights to exactly 0 (feature selection). The C parameter controls regularisation strength: C = 1/λ, where small C means strong regularisation and large C means weak regularisation.

10. Multi-Class Extension

Analogy: Binary classification is like choosing between two restaurants (Chinese or Italian). Multi-class is like choosing among five. Two strategies: (1) Run a series of head-to-head votes (One-vs-Rest) — "Is it Chinese vs. not-Chinese?", "Is it Italian vs. not-Italian?", etc. (2) Have all five restaurants simultaneously present their case and pick the strongest (Softmax).

Mapping:
  • Head-to-head vote → One-vs-Rest (OvR) with K binary classifiers
  • Simultaneous presentation → Softmax function
  • Each restaurant's "strength" → ezₖ for class k
  • "Pick the strongest" → argmax of softmax output

One-vs-Rest (OvR) Strategy

Definition — One-vs-Rest (OvR): For K classes, train K separate binary classifiers. Classifier k treats class k as "positive" and all other classes as "negative." At prediction time, the class with the highest predicted probability wins.
  • For K classes, you need K classifiers
  • Each classifier outputs P(y=k | x)
  • Prediction: argmax_k P(y=k | x)
  • Simple to implement — just use standard binary logistic regression K times
Worked Example — OvR for Fruit Classification (3 classes):

Classes: Apple (0), Banana (1), Orange (2)

Train 3 binary classifiers:

  • Classifier 1: Apple vs. {Banana, Orange} → P(y=0|x) = 0.7
  • Classifier 2: Banana vs. {Apple, Orange} → P(y=1|x) = 0.2
  • Classifier 3: Orange vs. {Apple, Orange} → P(y=2|x) = 0.5

Prediction: argmax(0.7, 0.2, 0.5) = 0 → Apple

Softmax Function

Definition — Softmax Function:
P(y = k | x) = ezₖ / Σⱼ ezⱼ

where zₖ = wₖᵀx + bₖ is the raw score for class k, and the sum is over all K classes.

Properties of Softmax

  • Output sums to 1: Σₖ P(y=k|x) = 1 (valid probability distribution)
  • All outputs positive: ezₖ > 0 for all k
  • Argmax preserved: The class with the highest zₖ gets the highest probability
  • Temperature parameter: Softmax(z/T) — T→0 makes it hard argmax, T→∞ makes it uniform
Worked Example — Softmax Calculation (3 classes):

Raw scores: z = [2.0, 1.0, 0.5]

Step 1: Compute exponentials

e2.0 = 7.389
e1.0 = 2.718
e0.5 = 1.649

Step 2: Compute sum

Σ = 7.389 + 2.718 + 1.649 = 11.756

Step 3: Divide each by sum

P(y=0|x) = 7.389 / 11.756 = 0.629
P(y=1|x) = 2.718 / 11.756 = 0.231
P(y=2|x) = 1.649 / 11.756 = 0.140

Check: 0.629 + 0.231 + 0.140 = 1.000 ✓

Prediction: class 0 with 62.9% confidence.

When K=2, Softmax = Sigmoid: For binary classification with two classes, softmax simplifies to sigmoid. If z₁ = z and z₂ = 0 (as the reference), then P(y=1|x) = ez/(ez+1) = 1/(1+e−z) = σ(z). This is why logistic regression is a special case of softmax regression.

Multi-Label vs Multi-Class

Common Confusion:
  • Multi-class: Each sample belongs to exactly ONE of K classes. Use softmax (outputs sum to 1).
  • Multi-label: Each sample can belong to MULTIPLE classes simultaneously. Use independent sigmoids for each class (outputs don't sum to 1).
  • Example: Photo tagging (a photo can have "dog", "outdoor", AND "sunny") → multi-label → independent sigmoids.
  • Example: Handwritten digit recognition (each image is exactly one digit) → multi-class → softmax.
Key Takeaway: For multi-class problems, use One-vs-Rest (K binary classifiers) or Softmax (K scores normalized to probabilities). Softmax ensures outputs sum to 1 and preserves the argmax. When K=2, softmax reduces to sigmoid. For multi-label problems, use independent sigmoids instead of softmax.

11. Common Mistakes

Mistake 1: Confusing P(A|B) with P(B|A)

The Confusion:
P(disease | positive test) ≠ P(positive test | disease)

P(positive test | disease) = sensitivity = 0.95 (95% of diseased test positive)

P(disease | positive test) = precision = ??? (depends on disease prevalence!)

Worked Example — The Fallacy:

A disease affects 1 in 1000 people. A test has 95% sensitivity and 95% specificity.

Out of 100,000 people:

  • 100 have the disease → 95 test positive (true positives)
  • 99,900 don't have the disease → 99,900 × 0.05 = 4,995 test positive (false positives)
P(disease | positive test) = 95 / (95 + 4995) = 95 / 5090 ≈ 1.87%

Despite the test seeming great (95% accuracy), a positive result only means 1.87% chance of disease! This is Bayes' theorem in action.

Mistake 2: Not Applying Feature Scaling

The Problem: Logistic regression uses gradient descent, which is sensitive to feature scales. If one feature ranges [0, 1000] and another [0, 1], the gradient will be dominated by the larger feature, causing:
  • Extremely slow convergence
  • Overshooting in the large-scale dimension
  • Weights that don't reflect true importance
Solution: Always standardise features (z-score: (x − μ)/σ) or normalise (min-max scaling) before training logistic regression.

Mistake 3: Using Sigmoid for Multi-Label

The Problem: Using softmax (outputs sum to 1) for multi-label classification forces the model to choose — if it increases P(class A), it must decrease P(class B), even if both should be true.

Solution: For multi-label tasks, use independent sigmoids for each class. Each class gets its own output neuron with σ, and the threshold (e.g., 0.5) is applied independently to each.

Mistake 4: Ignoring Class Imbalance

The Problem: If 99% of emails are not spam, a model that always predicts "not spam" achieves 99% accuracy but is useless.

Solutions:

  • Use class weights (penalize errors on minority class more heavily)
  • Resample: oversample minority, undersample majority
  • Use appropriate metrics: F1-score, AUC-ROC, not accuracy

Mistake 5: Interpreting Coefficients as Probabilities

The Problem: A weight w₁ = 0.8 does NOT mean the probability increases by 80%. It means the log-odds increase by 0.8, or equivalently, the odds are multiplied by e0.8 = 2.23. The actual probability change depends on the current probability.
Key Takeaway: Common mistakes include confusing conditional probabilities, skipping feature scaling, using the wrong activation for multi-label, ignoring class imbalance, and misinterpreting coefficients. Always check: Is this P(A|B) or P(B|A)? Are features scaled? Am I using sigmoid or softmax correctly?

Logistic Regression

What is it?

A probabilistic classification model that predicts P(y=1|x) by passing a linear score through the sigmoid function. Despite its name, it is a classification (not regression) algorithm. It models the log-odds as a linear function of the features.

Sigmoid Function

What is σ(z) and its key property?

σ(z) = 1 / (1 + e−z)

Maps any real number z ∈ (−∞, +∞) to a value in (0, 1). S-shaped, symmetric about (0, 0.5), with maximum gradient of 0.25 at z=0. Used to convert linear scores into probabilities.

Decision Boundary

What is it and where does it occur?

The hyperplane where the model switches its prediction. For logistic regression, it occurs where σ(z) = 0.5, i.e., where z = wᵀx + b = 0. Points on one side → class 1; points on the other → class 0. For 2D features, this is a line.

Odds

What are odds and how do they relate to probability?

Odds = P / (1 − P)

Odds range from 0 to ∞. Odds = 1 means P = 0.5. Odds = 3 means 3:1 (75% probability). Odds < 1 means the event is unlikely. Odds are multiplicative: an odds ratio of 2.23 means the odds double and then some.

Logit

What is the logit function?

logit(P) = ln(P / (1 − P))

The inverse of the sigmoid. Maps probabilities in (0, 1) to log-odds in (−∞, +∞). Logistic regression is linear in log-odds: logit(P) = wᵀx + b.

Binary Cross-Entropy (BCE)

What is the BCE loss formula?

L_BCE = −(1/N) Σ [y·log(ŷ) + (1−y)·log(1−ŷ)]

Derived from Maximum Likelihood Estimation. Convex loss function with a single global minimum. Penalizes confident wrong predictions heavily (loss → ∞ as ŷ → 0 for y=1).

Maximum Likelihood Estimation (MLE)

What is the core idea?

Find the parameter values (w, b) that maximize the probability of the observed training data. MLE for logistic regression leads to BCE loss. Key steps: write likelihood as product → take log (product→sum) → negate (max→min).

L1 Regularisation (Lasso)

What does it do differently from L2?

L = L_BCE + λ·Σ|wᵢ|

Pushes weights to exactly 0, performing automatic feature selection. Creates sparse models. Useful when many features are irrelevant. The L1 penalty is not differentiable at 0, but subgradient methods handle this.

L2 Regularisation (Ridge)

How does it prevent overfitting?

L = L_BCE + λ·Σwᵢ²

Penalizes large weights by adding the sum of squared weights. Shrinks all weights toward 0 but rarely makes them exactly 0. Encourages small, distributed weights rather than reliance on a few large weights.

Softmax Function

What does it do for multi-class?

P(y=k|x) = ezₖ / Σⱼ ezⱼ

Converts K raw scores into a valid probability distribution (all positive, sum to 1). Preserves the argmax. When K=2, softmax reduces to sigmoid: σ(z) = ez/(ez+1).

Cross-Entropy

What does it measure?

Cross-entropy measures the divergence between two probability distributions: the true distribution (y) and the predicted distribution (ŷ). Lower cross-entropy means better predictions. BCE is the binary case. For multi-class, use categorical cross-entropy.

Convex Loss

Why is it important for optimisation?

A convex function has exactly one global minimum and no local minima. Gradient descent is guaranteed to converge to the global minimum (with appropriate learning rate). BCE loss is convex in w, while MSE + sigmoid is non-convex (local minima trap gradient descent).

Scenario: Hospital Readmission Prediction

A hospital wants to predict whether a patient will be readmitted within 30 days. They train a logistic regression model with three features:

  • x₁ = age (years)
  • x₂ = number of previous admissions
  • x₃ = length of stay (days)

The trained model gives:

z = 0.05 × age + 0.8 × previous_admissions − 0.03 × stay − 2.5

Patient A: age=65, previous_admissions=3, stay=5 days

Patient B: age=45, previous_admissions=0, stay=2 days

Patient C: age=70, previous_admissions=5, stay=10 days

Task: For each patient, compute z, then σ(z), then classify (threshold = 0.5). Then interpret the weight for "previous admissions" as an odds ratio. Finally, identify which patient is at highest risk and explain why in terms of the model.

Click for full solution

Patient A: z = 0.05(65) + 0.8(3) − 0.03(5) − 2.5 = 3.25 + 2.4 − 0.15 − 2.5 = 3.0

σ(3.0) = 1/(1+e−3.0) = 1/(1+0.0498) = 1/1.0498 ≈ 0.953

P(readmit) = 95.3% → Class 1 (will be readmitted)

Patient B: z = 0.05(45) + 0.8(0) − 0.03(2) − 2.5 = 2.25 + 0 − 0.06 − 2.5 = −0.31

σ(−0.31) = 1/(1+e0.31) = 1/(1+1.363) = 1/2.363 ≈ 0.423

P(readmit) = 42.3% → Class 0 (will not be readmitted)

Patient C: z = 0.05(70) + 0.8(5) − 0.03(10) − 2.5 = 3.5 + 4.0 − 0.3 − 2.5 = 4.7

σ(4.7) = 1/(1+e−4.7) = 1/(1+0.0091) = 1/1.0091 ≈ 0.991

P(readmit) = 99.1% → Class 1 (will be readmitted)

Odds Ratio for previous admissions:

e0.8 = 2.2255

Each additional previous admission multiplies the odds of readmission by 2.23. This is the strongest predictor in the model.

Highest risk: Patient C (99.1%) because they have the most previous admissions (5 × 0.8 = 4.0 contribution to z), which dominates all other features. The model heavily penalizes repeat admissions, consistent with the medical intuition that frequent admissions signal chronic conditions.
