1. Why CNNs? – Images Break Fully-Connected Networks

Analogy: Imagine being asked to identify a friend's face, but someone smashes the photo into a long strip of 120,000 tiny pieces and scatters them on a table. You have no idea which piece was next to which. That is what happens when you flatten an image and feed it to a fully-connected network — you destroy all spatial structure.

A 200×200 colour image has 200 × 200 × 3 = 120,000 pixel values. If the first hidden layer has just 1,000 neurons, you need 120,000,000 weights for that single layer. This is:

The Parameter Explosion Problem
• Massive memory & compute cost.
• Extreme overfitting — far more parameters than training samples.
• No guarantee the network learns anything useful.

Two More Critical Failures

No spatial awareness: Flattening treats pixel (0,0) and pixel (199,199) as equally related to every hidden neuron. The network must re-learn which pixels are neighbours — an enormously wasteful task.

Not translation invariant: If a cat appears in the top-left of training images but the centre of test images, a fully-connected network has never seen it in that position and will likely fail.

CNN Insight — Three Principles:
1. Local connectivity: Each neuron looks only at a small patch of the input.
2. Weight sharing: The same small set of weights (a filter) is reused at every position.
3. Translation equivariance: If the input shifts, the output feature map shifts by the same amount — the network doesn't care where a feature is.
Key Takeaway: CNNs replace the "see everything at once" approach of fully-connected layers with a "scan with a small stamp" approach — massively reducing parameters while preserving spatial structure.

2. The Coarse-to-Fine Hierarchy

Analogy — How You Recognise Faces: When you look at a person, your brain first registers basic edges and contrasts. Then it assembles those into features: "there's an eye here, a nose there." Only then does it combine those features into "that's a face" and "that's my friend." You never jump straight from raw pixels to identity — it's a hierarchy.

A CNN mirrors this process exactly:

The Hierarchy:
Layer 1 (Conv1): Detects simple edges — horizontal, vertical, diagonal.
Layer 2 (Deeper Conv): Combines edges into textures, corners, and small patterns (an eye, a nostril).
Layer 3+ (Even Deeper): Assembles parts into whole objects — a face, a car, a dog.
Dense + Classifier: Maps the high-level representation to a class label.
Representation Learning: The CNN does not require you to hand-engineer which features matter. It automatically discovers the best intermediate representations (edges → parts → objects) through gradient descent on the task loss.
Key Takeaway: The power of CNNs is that they learn a hierarchy of increasingly abstract features — from raw pixels all the way to semantic concepts — entirely from data.

3. Filters (Kernels)

Analogy — A Rubber Stamp: Imagine you have a small rubber stamp painted with a pattern (say, a horizontal line). You press that stamp all over the page, sliding it one cell at a time. Every time you stamp, you note how well the pattern "matches" the ink underneath. The result is a new sheet that highlights everywhere the horizontal line appeared. That is exactly what a CNN filter does.

A filter (also called a kernel) is a small matrix of weights — typically 3×3 or 5×5 — that slides across the input image.

How it works:
1. Pick a pattern: The filter encodes a pattern (e.g. "horizontal edge") via its weights.
2. Slide across the image: At each position, overlay the filter on the corresponding image patch.
3. Record match scores: Multiply element-wise, sum, add bias → single number per position.
4. The resulting grid is called a feature map.

Horizontal Edge Detector Example

K_horizontal =
┌ ┐
│ 1 1 1 │
│ 0 0 0 │
│ -1 -1 -1 │
└ ┘

This filter gives a high positive score when the top rows are bright and bottom rows are dark (a horizontal edge), and a negative score for the reverse.

Weight Sharing: The same filter is applied at every position. A 3×3 filter has only 9 weights + 1 bias = 10 parameters, regardless of image size. This is the key to CNN efficiency.
Key Takeaway: A filter is a tiny, reusable pattern detector. It slides across the image to produce a feature map showing where that pattern appears.

4. Images as Grids of Numbers

Before we can do any math, we need to understand how images are represented.

Greyscale Image

A greyscale image is a single 2D grid of numbers (one channel):

Shape: H × W × 1
Each value: 0 (black) to 255 (white)

Example — a 4×4 greyscale patch:

┌ ┐
│ 50 80 120 200 │
│ 60 90 130 180 │
│ 70 100 140 160 │
│ 40 70 110 150 │
└ ┘

Colour (RGB) Image

A colour image has three channels stacked along the depth axis:

Shape: H × W × 3
Channel 0: Red values
Channel 1: Green values
Channel 2: Blue values
Common Mistake: Students often think the "×3" means each pixel is a 3-element vector. It's more accurate to think of it as three separate 2D grids stacked on top of each other — like three sheets of graph paper, one for each colour.
Key Takeaway: Images are just numbers in a grid. Greyscale = H×W×1, Colour = H×W×3. The CNN operates directly on these numerical grids.

5. The Convolution Operation

The core operation of a CNN layer is (technically) cross-correlation, though the community calls it "convolution."

Z(i, j) = Σm Σn I(i+m, j+n) · K(m, n) + b

Where:

  • I = input image (or feature map)
  • K = filter/kernel matrix
  • b = scalar bias
  • Z = output feature map
  • (i, j) = position in the output feature map
Feature Map: The 2D grid produced by sliding one filter across the entire input. Each cell is a match score: "how much does this filter's pattern exist at this position?"
Receptive Field: The region of the input that a single output neuron "sees." For a 3×3 filter, each output neuron has a 3×3 receptive field — it only knows about those 9 input values (per channel).
Weight Sharing: The same filter weights K(m,n) and bias b are used at every output position. Position (0,0) and position (5,5) use the exact same 9 weights. This is what makes CNNs parameter-efficient.
Key Takeaway: Convolution = place the filter at each position, element-wise multiply with the input patch, sum everything, add bias. Result = one cell of the feature map.

6. Stride, Padding, and Output Size

Stride (S): How many pixels the filter moves at each step. Stride=1 → move 1 pixel. Stride=2 → skip every other position (halves the output size approximately).
Padding (P): Zeros added around the border of the input. Padding=1 adds a 1-pixel ring of zeros, allowing the filter to process edge pixels and control the output size.
O = ⌊(W − F + 2P) / S⌋ + 1

Where:

  • O = output spatial dimension
  • W = input spatial dimension
  • F = filter size
  • P = padding
  • S = stride

Worked Examples

W F P S Calculation O
5 3 0 1 ⌊(5−3+0)/1⌋+1 = 2+1 3
32 5 0 1 ⌊(32−5+0)/1⌋+1 = 27+1 28
32 5 2 1 ⌊(32−5+4)/1⌋+1 = 31+1 32 (same padding)
7 3 0 2 ⌊(7−3+0)/2⌋+1 = 2+1 3
Analogy — "Same" Padding: Think of "same" padding as putting a frame around a painting so the frame can be measured too. Without the frame (padding=0), the edges of the painting are measured less often than the centre. With a frame, every pixel gets equal attention, and the output size stays the same as the input.
Key Takeaway: The output formula O = ⌊(W−F+2P)/S⌋+1 controls spatial dimensions. Larger stride → smaller output. More padding → larger output. "Same" padding keeps output = input.

8. ReLU on the Feature Map

After convolution, we apply an activation function. The most common is ReLU (Rectified Linear Unit).

ReLU(Z) = max(0, Z) — applied element-wise

Applied to our feature map from Section 7:

Z: ReLU(Z):
2 -3 -4 → 2 0 0
-1 0 -3 → 0 0 0
-1 2 2 → 0 2 2

Three cells are now zero (were negative or zero).

Why ReLU?

  • Adds non-linearity: Without it, stacking conv layers is just one big linear operation — no more expressive than a single layer.
  • Computationally cheap: Just a single comparison per element — no exponentials like sigmoid or tanh.
  • Sparse activation: Many zeros in the output. This is biologically plausible and computationally efficient.
Dead ReLU Risk: If a neuron always receives negative input (its weights update to always produce negatives), ReLU always outputs 0. That neuron is "dead" — it never contributes and its gradient is always 0, so it can never recover. We'll cover this in depth in Section 11.
Key Takeaway: ReLU turns all negative values to zero while passing positives through unchanged. It adds the non-linearity the network needs, and is cheap to compute.

9. Pooling

Pooling down-samples a feature map, reducing its spatial dimensions while keeping the most important information.

Max Pooling

Slide a window across the feature map and output the maximum value in each region.

Input (3×3) after ReLU:
┌ ┐
│ 2 0 0 │
│ 0 0 0 │
│ 0 2 2 │
└ ┘

2×2 Max Pool, Stride = 1:

P(0,0) = max(2, 0, 0, 0) = 2
P(0,1) = max(0, 0, 0, 0) = 0
P(1,0) = max(0, 0, 0, 2) = 2
P(1,1) = max(0, 0, 2, 2) = 2

Output (2×2):
┌ ┐
│ 2 0 │
│ 2 2 │
└ ┘

Output size: O = ⌊(3 − 2) / 1⌋ + 1 = 1 + 1 = 2

Average Pooling

Instead of max, take the mean of each region. Less common in modern architectures but used in some designs.

Benefits of Pooling:
1. Down-sampling: Reduces spatial dimensions → fewer parameters in subsequent layers.
2. Translation invariance: Small shifts in the input don't change the max/average much → more robust features.
3. Extracts dominant features: Keeps the strongest signal in each region.
Key Takeaway: Pooling reduces spatial size while retaining the strongest features. Max pooling keeps the most prominent activation; average pooling keeps the general trend.

10. Complete Convolutional Block — End to End

Let's trace a single convolutional block from start to finish with real numbers.

INPUT: 5×5 image
┌ ┐
│ 2 1 3 0 1 │
│ 1 2 1 1 0 │
│ 0 1 2 3 2 │
│ 3 0 1 2 1 │
│ 1 2 0 1 3 │
└ ┘

STEP 1 — CONV: 3×3 horizontal edge filter, bias = −1, stride = 1, no padding
→ 3×3 feature map:
2 -3 -4
-1 0 -3
-1 2 2

STEP 2 — ReLU: max(0, ·) element-wise
→ 3×3 activated map:
2 0 0
0 0 0
0 2 2

STEP 3 — MAX POOL (2×2, stride 1):
→ 2×2 output:
2 0
2 2
Full Chain Summary:
5×5 input → [Conv 3×3] → 3×3 feature map → [ReLU] → 3×3 activated → [Pool 2×2] → 2×2 output

The spatial dimensions shrunk from 5×5 to 2×2, but the output now encodes "where horizontal edges are" — a meaningful high-level feature.
Key Takeaway: A conv block chains three operations: Convolution (extract features), ReLU (add non-linearity), Pooling (down-sample). Each block progressively transforms raw pixels into meaningful features.

11. Dead ReLU in CNNs

Dead ReLU is a common training pathology where a neuron permanently outputs zero and stops learning.

How It Happens

Healthy neuron: Cell value = 2
→ ReLU output = 2 (positive, gradient = 1)
→ Backprop: gradient flows through, weights update

Dead neuron: Cell value = −3
→ ReLU output = 0 (clamped, gradient = 0)
→ Backprop: gradient is zero, weights receive NO update
→ Next forward pass: still negative → still zero → permanently dead

If a filter's weights drift such that it always produces negative values for all training inputs, it can never recover because the gradient is always zero.

Why This Matters: A network with many dead neurons has reduced capacity — it's effectively a smaller network. If too many neurons die, training stalls.

Mitigation Strategies

Solutions:
1. Leaky ReLU: Instead of 0 for negatives, use f(x) = 0.01x for x < 0. Small gradient always flows.
2. Lower learning rate: Prevents weights from overshooting into permanently-negative territory.
3. He initialization: Initialize weights from N(0, √(2/n_in)) — keeps initial activations centred around zero.
Leaky ReLU(x) = { x, if x > 0; αx, if x ≤ 0 } where α = 0.01
Key Takeaway: Dead ReLU occurs when neurons get stuck outputting zero with zero gradient. Leaky ReLU, careful initialization, and moderate learning rates prevent this.

12. Hierarchical Feature Learning

One of the most remarkable properties of CNNs is that they automatically learn a hierarchy of features — no manual feature engineering required.

Analogy — Painting a Portrait: A painter starts with broad strokes (background, overall shape), then adds features (eyes, nose, mouth), then fine details (pupils, eyelashes). Each layer of detail builds on the previous. A CNN does the same: coarse features feed into finer features.

The Feature Hierarchy

Layer 1: Edges and lines — horizontal, vertical, diagonal, gradients
Layer 2: Textures and corners — repeated edge patterns, blob detectors, corner detectors
Layer 3: Parts — eyes, wheels, ears, letters
Layer 4: Whole objects — faces, cars, animals, words

This is learned automatically through loss minimization. The network discovers that detecting edges is useful because it helps detect corners, which helps detect eyes, which helps detect faces, which minimises the classification loss.

Key Takeaway: CNNs learn features in a hierarchy — from low-level (edges) to high-level (objects) — entirely through end-to-end training on the task loss.

13. Complete CNN Architecture

A full CNN has two main parts: a feature extractor and a classifier head.

FEATURE EXTRACTOR (Conv/Pool Stack):

Input Image → [Conv → ReLU → Pool] → [Conv → ReLU → Pool] → ... → Feature Tensor

Each block extracts increasingly abstract features.

CLASSIFIER HEAD:

Feature Tensor → [Flatten] → [Dense] → [Softmax] → Class Probabilities

Example: Simple CNN for 32×32 Images, 10 Classes

Layer Operation Output Shape Params
Input Image 32 × 32 × 3 —
Conv1 3×3 filter, 8 filters, ReLU 30 × 30 × 8 8 × (3×3×3 + 1) = 224
Pool1 2×2 max pool, stride 2 15 × 15 × 8 0
Conv2 3×3 filter, 16 filters, ReLU 13 × 13 × 16 16 × (3×3×8 + 1) = 1,168
Pool2 2×2 max pool, stride 2 6 × 6 × 16 0
Flatten Reshape to vector 576 0
Dense Fully-connected, ReLU 64 576 × 64 + 64 = 36,928
Output Softmax 10 64 × 10 + 10 = 650
Key Takeaway: A CNN architecture is a feature extractor (conv/pool blocks) followed by a classifier (flatten → dense → softmax). The entire network is trained end-to-end by backpropagation.

14. Weight Sharing — Parameter Counting

Weight sharing is the fundamental reason CNNs are so much more efficient than fully-connected networks.

Fully-Connected vs. Convolutional

Fully-Connected: 5×5 input → 3×3 output
Flattened input: 25 neurons. Output: 9 neurons.
Weights: 25 × 9 = 225 weights + 9 biases = 234 parameters

Convolutional: Same task with a 3×3 filter
One 3×3 filter = 9 weights + 1 bias = 10 parameters
(And this filter works for ANY input size!)
Parameters per filter = (F × F × Cin + 1) × Cout

Where:

  • F = filter spatial size
  • Cin = number of input channels
  • Cout = number of output filters (output channels)
  • +1 = bias per filter

Example: 3×3 filters, 3 input channels, 16 output feature maps

Per filter: (3 × 3 × 3 + 1) = 28 parameters
Total: 28 × 16 = 448 parameters

Compare: the same spatial operation in a fully-connected layer would need:
(H × W × 3) × (H' × W' × 16) parameters — potentially millions.
Key Takeaway: Weight sharing means one filter has (F² × Cin + 1) parameters regardless of image size. A fully-connected layer with the same connectivity would need orders of magnitude more.

15. Common Calculation Mistakes

These are the errors students make most often on exams and in practice:

Mistake 1: Wrong output size (forgetting +1)
Students compute ⌊(W−F+2P)/S⌋ and forget to add 1.
Correct: O = ⌊(5−3)/1⌋ + 1 = 2 + 1 = 3, not 2.
Mistake 2: Forgetting the bias
Students compute only the sum of element-wise products and omit the bias term.
Correct: Z(i,j) = Σ Σ (I·K) + b
Mistake 3: Applying ReLU before the sum
Students apply ReLU to individual products before summing.
Correct: First compute the full weighted sum + bias, THEN apply ReLU to the result.
Mistake 4: Pooling stride confusion
Students use the pool window size as the stride (always) or forget to apply the output size formula to pooling.
Correct: For 2×2 pool with stride 1 on a 3×3 input: O = ⌊(3−2)/1⌋ + 1 = 2, not 1.
Mistake 5: Flipping the filter
In mathematically true convolution, the filter is flipped before sliding. In CNN practice, we do cross-correlation (no flip). Students sometimes flip K and produce wrong results.
Correct: In deep learning, just overlay and multiply — do NOT flip.
Mistake 6: Miscounting zeros after ReLU
Students count zeros incorrectly — sometimes missing that 0 is also clamped (ReLU(0) = 0), or counting negatives as if they passed through.
Correct: ReLU(x) = 0 for ALL x ≤ 0, including zero itself.
Key Takeaway: The six most common mistakes are: forgetting +1, dropping the bias, ReLU before sum, pool stride confusion, flipping the filter, and miscounting ReLU zeros. Double-check each step systematically.

Flashcards

CNN (Convolutional Neural Network)
A neural network that uses convolution operations with learnable filters to process grid-structured data (like images), exploiting local connectivity, weight sharing, and translation equivariance.
Convolution
Sliding a filter across the input, performing element-wise multiplication with the local patch, summing all products, and adding a bias. Produces one value per position in the output feature map. (Technically cross-correlation in DL.)
Filter / Kernel
A small matrix of learnable weights (typically 3×3 or 5×5) that detects a specific pattern (e.g. horizontal edge). It is slid across the entire input to produce a feature map.
Receptive Field
The region of the input that a single output neuron "sees" or depends on. For a 3×3 filter, each output neuron has a 3×3 receptive field — it only uses values from that patch.
Stride
The number of pixels the filter moves at each step. Stride=1 means the filter shifts one pixel at a time (full overlap). Stride=2 means it skips every other position, reducing output size by roughly half.
Padding
Zeros added around the border of the input before convolution. Padding allows edge pixels to be centred in the filter, and "same" padding (P=(F−1)/2) keeps the output size equal to the input size.
Feature Map
The 2D output produced by applying one filter across the entire input. Each cell is a match score indicating how strongly the filter's pattern is present at that spatial position.
Max Pooling
Slides a window (e.g. 2×2) across a feature map and outputs the maximum value in each region. Down-samples the spatial dimensions while keeping the most prominent activation.
Weight Sharing
The same filter (same weights and bias) is applied at every spatial position. A 3×3 filter always has 9 weights + 1 bias = 10 parameters, regardless of image size. This dramatically reduces parameter count.
Translation Equivariance
If the input shifts by some amount, the output feature map shifts by the same amount. The CNN detects the same features regardless of where they appear in the image.
Pooling
1. Down-sampling — reduces spatial size for efficiency.
2. Translation invariance — small shifts don't change the output much.
3. Dominant feature extraction — keeps the strongest signal per region.
Feature Hierarchy
Layer 1: edges and lines. Layer 2: textures and corners. Layer 3: object parts (eyes, wheels). Layer 4: whole objects (faces, cars). Learned automatically through end-to-end training.

Scenarios

Scenario 1: Designing a CNN for Medical Imaging

Scenario: You are building a CNN to classify 512×512 greyscale X-ray images as "healthy" or "pneumonia." You need to choose between:

Option A: A fully-connected network with one hidden layer of 500 neurons.
Option B: A CNN with 3 conv/pool blocks followed by a dense classifier.

Questions:
1. How many input parameters does Option A need for the first layer?
2. Why is Option B almost certainly better?
3. What type of features would you expect the CNN to learn at each layer?
Solution:

1. Input: 512 × 512 × 1 = 262,144 pixels. First hidden layer: 262,144 × 500 = 131,072,000 weights + 500 biases ≈ 131 million parameters. This is enormous and will severely overfit with typical medical image datasets (often only thousands of images).

2. Option B is better because:
• Conv filters share weights: a 3×3 filter = 10 params, reused across all positions.
• Spatial structure is preserved — edges, textures, shapes are detected based on their pattern, not their absolute position.
• Translation equivariance means pneumonia indicators are detected regardless of where they appear in the lung.
• Far fewer parameters → less overfitting on small medical datasets.

3. Expected features:
• Block 1: Edges and gradients (bone boundaries, lung tissue edges)
• Block 2: Textures (opacity patterns, consolidation regions)
• Block 3: Disease-specific patterns (ground-glass opacities, nodules)
• Dense layer: Combines these to classify healthy vs. pneumonia.

Scenario 2: Debugging a Failing CNN

Scenario: Your CNN trained on CIFAR-10 (32×32 colour images) is showing 90% training accuracy but only 45% test accuracy. You inspect the convolutional layers and find that 60% of ReLU outputs are exactly zero across all training samples.

Questions:
1. What problem does this suggest?
2. What are two things you could try to fix it?
3. How would you verify the fix worked?
Solution:

1. Massive dead ReLU problem: 60% of neurons are permanently dead — they always output zero. This means 60% of the network's capacity is wasted. These neurons receive no gradient updates and never recover. This could be caused by too-high learning rate, bad initialization, or too-aggressive training.

2. Fixes:
• Switch to Leaky ReLU (α=0.01): negatives get a small gradient, so dead neurons can recover.
• Reduce the learning rate (e.g. from 0.01 to 0.001): prevents weights from overshooting into permanently-negative territory.
• Apply He initialization: initialize weights from N(0, √(2/fan_in)) to keep initial activations centred around zero.

3. Verification:
• Monitor the percentage of zero activations per layer during training — it should stay below 10-20%.
• Check if test accuracy improves (gap between train and test should narrow).
• After training, count ReLU zeros: if still high, further tune learning rate or initialization.
