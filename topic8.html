8.1 Bridge from Week 7

Last week we built the foundation: a Multi-Layer Perceptron (MLP) is a stack of linear transformations interspersed with activation functions. Let's quickly recap the key pieces before we extend them into deep architectures.

The MLP in One Sentence

An MLP computes a series of transformations from input to output. Each layer performs a weighted sum, adds a bias, and then applies a non-linear activation function:

z(l) = W(l) · a(l−1) + b(l)     then     a(l) = φ(z(l))

Where a(0) = x (the input), W(l) is the weight matrix for layer l, b(l) is the bias vector, and φ is the activation function.

Forward Pass Recap

Data flows forward through the network: input → layer 1 → layer 2 → ... → output. At each layer we compute the pre-activation z, then the post-activation a. The final output ŷ = a(L) is the network's prediction.

Analogy: Think of the forward pass as an assembly line in a factory. Raw materials (input) enter at one end. Each station (layer) transforms the product — cutting, painting, assembling. At each station, two things happen: the product is measured and modified (weighted sum + bias), then inspected and approved or rejected (activation). The finished product (output) emerges at the end.

Backpropagation Recap

Training works by computing the loss, then propagating the error backward through the network using the chain rule. For each layer, we compute:

  1. The error signal δ(l) = ∂L/∂z(l) (how much the loss changes with respect to the pre-activation)
  2. The weight gradient ∂L/∂W(l) = δ(l) · (a(l−1))T
  3. The bias gradient ∂L/∂b(l) = δ(l)
  4. Update: W ← W − η · ∂L/∂W
Key Insight: Backpropagation is just the chain rule applied repeatedly. The gradient at layer l depends on the gradient at layer l+1, multiplied by the local derivative of the activation function at layer l. This means every activation function's derivative participates in the gradient chain. If any derivative is small, the entire chain shrinks.

Sigmoid: Week 7's Activation

In Week 7, we used the sigmoid function φ(z) = 1/(1 + e−z) in our MLP. Sigmoid squashes any real number into the range (0, 1), which is useful for binary classification output layers. But as we'll see, sigmoid has serious problems when used in hidden layers of deep networks.

σ(z) = 1 / (1 + e−z)     σ'(z) = σ(z) · (1 − σ(z))

Warning: The sigmoid derivative σ'(z) = σ(z)(1 − σ(z)) has a maximum value of 0.25 (when z = 0). For large |z|, the derivative approaches 0. This property — called saturation — is the root cause of the vanishing gradient problem, which we'll tackle next.
Key Takeaway: An MLP is a stack of linear transforms (z = Wa + b) with non-linear activations (a = φ(z)). Forward pass computes predictions; backpropagation propagates error backward using the chain rule. Every activation derivative multiplies into the gradient chain — choosing the right activation function is critical for deep networks.
Recap: MLP = layers of z = Wa + b followed by a = φ(z). Forward pass: input to output. Backprop: loss to input via chain rule. Sigmoid works but has derivative ≤ 0.25, which causes problems in deep networks.

8.2 The Collapse Problem

Before discussing which activation to use, let's understand why activation functions matter at all. What happens if we remove them?

Two Layers Without Activation

Consider a two-layer network with no activation function (or equivalently, a linear activation φ(z) = z):

Layer 1: z₁ = W₁x + b₁    a₁ = z₁

Layer 2: z₂ = W₂a₁ + b₂ = W₂(W₁x + b₁) + b₂ = (W₂W₁)x + (W₂b₁ + b₂)

Let W' = W₂W₁ and b' = W₂b₁ + b₂

Result: ŷ = W'x + b' — a single linear map!

Analogy: Imagine you have two cameras stacked together. Camera 1 applies a zoom. Camera 2 applies a rotation. The combined effect is just one zoomed-and-rotated image. Stacking linear operations is still linear — you can't create curves by stacking straight lines, no matter how many you stack. You need a lens that distorts (a non-linearity) to break the pattern.

The Mathematical Proof (2 Layers)

With L layers, all linear:

ŷ = WL(WL−1(…(W₁x + b₁)… + bL−1) + bL

= (WL · WL−1 · … · W₁)x + (combined biases)

= Wtotal · x + btotal

No matter how many layers you stack, without non-linear activations, the entire network collapses to a single linear transformation. A 100-layer network with linear activations is mathematically identical to a single-layer network. All that extra computation buys you nothing.

Critical Insight: Activations are the only source of expressive power in a neural network. Without them, you're just doing linear regression with extra steps. The non-linearity is what allows the network to learn curved decision boundaries, complex mappings, and hierarchical features.

What Makes a Good Activation Function?

An ideal activation function should satisfy these properties:

PropertyWhy It Matters
Non-linearWithout this, the network collapses to a single linear map (as shown above).
DifferentiableBackpropagation needs gradients. The activation must have a well-defined derivative everywhere (or almost everywhere).
Computationally cheapActivations are applied to every neuron in every forward pass. Expensive functions (e.g., involving large exponentials) slow training dramatically.
Gradient-friendlyThe derivative should not vanish for typical input ranges. A gradient of 0 means the neuron can't learn. A gradient that's always 1 (like ReLU) ensures gradients flow freely.
Could we use sin(z) or z² as activations?

Technically, any non-linear function could work as an activation. sin(z) is periodic and smooth, and z² is simple and differentiable. However, sin(z) causes oscillating gradients that make training unstable, and z² grows without bound, causing exploding activations in deep networks. ReLU succeeds because it's simple, bounded on one side, and has a constant gradient on the positive side. Practical considerations like these are why certain activations dominate.

Key Takeaway: Without non-linear activations, a deep network collapses to a single linear transformation: W'x + b'. Activations are the ONLY source of expressive power. Good activations are non-linear, differentiable, computationally cheap, and have gradient-friendly derivatives.
Recap: Stacking linear layers without activation produces ŷ = W'x + b' — one linear map regardless of depth. Activations add non-linearity, enabling curved decision boundaries. A good activation is non-linear, differentiable, cheap, and gradient-friendly.

8.3 The Sigmoid Problem — Vanishing Gradients

We established that non-linear activations are essential. Sigmoid (σ) was one of the first widely used activations. But it has a致命 flaw that cripples deep networks: the vanishing gradient problem.

Sigmoid's Derivative Is Tiny

The sigmoid derivative is:

σ'(z) = σ(z) · (1 − σ(z))

This function is a bell-shaped curve centered at z = 0, with a maximum value of 0.25. For any input z, the derivative is always in the range [0, 0.25].

Analogy: Imagine you're playing a game of telephone (Chinese whispers). Message 1 starts clear. After each person whispers to the next, the message gets slightly muffled. By the time it reaches the 10th person, it's barely audible. That's what happens to gradients in a deep sigmoid network — each layer's derivative (≤ 0.25) mutes the gradient a little more, and after several layers, the gradient is essentially zero. The early layers hear nothing and can't learn.

Stacking Sigmoids: Gradient Shrinkage

In backpropagation through L layers with sigmoid activations, the gradient at layer 1 depends on the product of all intermediate derivatives:

∂L/∂W₁ ∝ σ'(z₁) · σ'(z₂) · … · σ'(zL−1) · δout

Each σ'(z) ≤ 0.25. So the product of L sigmoid derivatives is bounded by:

(0.25)L

For a 10-layer network: (0.25)10 ≈ 0.000000954 — less than one millionth of the original gradient!

20≈ 10−12Impossible — gradients are numerical noise
The Vanishing Gradient Problem: When gradients shrink toward zero as they propagate backward through many layers, the early layers (which learn low-level features like edges, textures) receive almost no learning signal. The network cannot train these layers effectively, regardless of how long you train. This is why deep sigmoid networks were considered impossible before modern activations.

Why This Happens: The Chain Rule Bottleneck

The chain rule multiplies derivatives across layers. When each factor is ≤ 0.25, the product decays exponentially with depth. It's like compound interest in reverse — instead of growing, the gradient shrinks multiplicatively at every step.

Why does saturation make gradients vanish?

"Saturation" means the activation function's output is stuck near its extreme values (0 or 1 for sigmoid). When σ(z) ≈ 0 or σ(z) ≈ 1, the derivative σ'(z) = σ(z)(1−σ(z)) ≈ 0. In deep networks, most neurons' inputs are far from 0 (they accumulate through layers), so most neurons are saturated. This means most gradient contributions are near zero. Even when z ≈ 0 (the maximum gradient of 0.25), stacking 10+ layers still kills the gradient. Sigmoid has no regime where its derivative is close to 1, so there's no "safe zone" for gradient flow.

Key Takeaway: Sigmoid's derivative is at most 0.25. Stacking L sigmoid layers produces a gradient shrinkage of (0.25)^L — exponentially small for deep networks. Early layers can't learn. This is the vanishing gradient problem, and it motivated the shift to ReLU.
Recap: σ'(z) ≤ 0.25 for all z. In deep networks, gradient ∝ ∏σ'(zi), which shrinks exponentially with depth. After 10+ layers, gradients are near zero. Early layers cannot learn. Solution: use activations with larger derivatives (e.g., ReLU).

8.4 ReLU Activation

The Rectified Linear Unit (ReLU) is the most widely used activation function in deep learning. It solved the vanishing gradient problem and enabled the training of networks with dozens or hundreds of layers.

Definition

ReLU(z) = max(0, z) = { z  if z > 0,  0  if z ≤ 0 }

That's it. ReLU simply outputs the input if it's positive, and outputs 0 otherwise.

Analogy: Think of ReLU as a bouncer at a club. Positive values get in (passed through unchanged). Negative values are turned away (output zero). Zero is right at the door — also turned away. There's no gradual fading or complex decision — it's a hard gate: positive or nothing. This simplicity is exactly what makes it work so well.

Why ReLU Works: Properties

PropertyValueWhy It Matters
Gradient for z > 01Constant gradient of 1 means gradients flow through unchanged — no vanishing!
Gradient for z < 00Dead zone — outputs and gradients are zero for negative inputs.
Range[0, ∞)Unbounded on the positive side — no saturation for positive inputs.
Computational costComparison + identityIncredibly cheap: just check if z > 0. No exponentials, no divisions.
Sparsity~50% of neurons output 0Sparse representations are more efficient and can improve generalization.
Why gradient = 1 matters: When the gradient is 1, multiplying by it during backpropagation doesn't shrink the gradient at all. If the loss gradient arriving at a ReLU neuron is δ, the gradient passed to the previous layer is also δ (for positive inputs). This means gradients can propagate through arbitrarily deep networks without shrinking — completely solving the vanishing gradient problem.

Worked Example: Forward Pass Through ReLU

Given input vector x = [2.0, −1.5, 3.0, −0.5, 0.0], compute ReLU(x) element by element:

Worked Example: ReLU Forward Pass
1 Element 1: x₁ = 2.0
2.0 > 0, so ReLU(2.0) = 2.0
2 Element 2: x₂ = −1.5
−1.5 ≤ 0, so ReLU(−1.5) = 0.0
3 Element 3: x₃ = 3.0
3.0 > 0, so ReLU(3.0) = 3.0
4 Element 4: x₄ = −0.5
−0.5 ≤ 0, so ReLU(−0.5) = 0.0
5 Element 5: x₅ = 0.0
0.0 ≤ 0, so ReLU(0.0) = 0.0
✓ Result
ReLU([2.0, −1.5, 3.0, −0.5, 0.0]) = [2.0, 0.0, 3.0, 0.0, 0.0]

ReLU Derivative

ReLU'(z) = { 1  if z > 0,  0  if z < 0,  undefined at z = 0 }

In practice, the derivative at z = 0 is set to 0 (or 1, depending on the framework). Since z = 0 is a single point, it almost never occurs in practice, so this doesn't cause issues.

Worked Example: ReLU Derivative
1 Derivative of ReLU(2.0)
2.0 > 0 → ReLU'(2.0) = 1. Gradient passes through unchanged.
2 Derivative of ReLU(−1.5)
−1.5 < 0 → ReLU'(−1.5) = 0. Gradient is killed — this neuron doesn't learn from this input.
3 Derivative of ReLU(0.0)
z = 0 → ReLU'(0) = 0 (by convention). Rarely encountered in practice.

Historical Note: AlexNet (2012)

Before 2012, sigmoid and tanh were the default activations. Alex Krizhevsky's AlexNet won the ImageNet competition in 2012 using ReLU activations, demonstrating dramatically faster training (6× speedup over sigmoid) and better performance. This single result triggered the deep learning revolution and made ReLU the standard activation for hidden layers.

Why was ReLU faster to train than sigmoid?

Three reasons: (1) Computation — ReLU is a simple comparison (if z > 0, output z), while sigmoid requires an exponential and a division. For millions of neurons, this adds up. (2) Gradient flow — ReLU's gradient of 1 for positive inputs means gradients don't shrink, so early layers learn quickly. With sigmoid, early layers get vanishingly small gradients and learn very slowly. (3) Sparsity — ReLU outputs many zeros, creating sparse activations that are computationally efficient and can improve generalization by reducing redundancy.

Warning: ReLU is NOT suitable for output layers in most tasks. For regression, use a linear output (no activation). For binary classification, use sigmoid. For multi-class, use softmax. ReLU is for hidden layers only.
Key Takeaway: ReLU(z) = max(0, z). Gradient = 1 for z > 0 (no vanishing), gradient = 0 for z < 0 (dead zone). Computationally trivial, produces sparse activations. Popularized by AlexNet 2012. Use ReLU in hidden layers; use task-specific activations in output layers.
Recap: ReLU(z) = max(0, z). Gradient is 1 when active, 0 when inactive. No saturation for positive inputs → gradients flow freely through deep networks. Cheap to compute. AlexNet 2012 made it the standard. Use in hidden layers, not output layers.

8.5 The Dying ReLU Problem

ReLU's simplicity has a downside: neurons can "die" — getting stuck in the negative region where the gradient is always 0, never updating again.

How a Neuron Dies

When a neuron's weighted sum z = Wx + b is negative for ALL training examples, the neuron outputs 0 for everything. Its gradient is also 0 (since ReLU'(z) = 0 for z < 0). With zero gradient, the weights never update. The neuron is permanently dead.

Analogy: Imagine a light switch that only turns on when voltage is positive. If the voltage drops below zero, the switch turns off and the bulb goes dark. Now imagine the switch also disables its own wiring when it turns off — no current can flow to repair it. Once off, it stays off forever. That's a dying ReLU neuron: negative input → zero output → zero gradient → no weight update → stays negative forever.

Common Causes

CauseWhy It Kills Neurons
Large negative bias initializationIf b is initialized very negative, z = Wx + b is negative for most inputs → neuron starts dead and never recovers.
High learning rateA large weight update can push z from slightly positive to deeply negative, killing the neuron in one step.
Poor weight initialisationWeights that are too large or too small can cause most neurons to output 0 initially, leading to widespread neuron death.
Data with many negative pre-activationsIf the data distribution naturally produces negative z values for many neurons, those neurons will be permanently inactive.
Warning: In practice, with proper initialisation (He init) and moderate learning rates, only a small percentage of ReLU neurons die. But with poor hyperparameters, up to 50% of neurons can become permanently dead — effectively halving your network's capacity.

ReLU Variants: Solving the Dying Problem

Leaky ReLU

LeakyReLU(z) = { z  if z > 0,  αz  if z ≤ 0 }     (typically α = 0.01)

Instead of outputting 0 for negative inputs, Leaky ReLU outputs a small fraction (αz). This ensures the gradient is always non-zero (α for negative inputs, 1 for positive), so neurons can always learn.

ELU (Exponential Linear Unit)

ELU(z) = { z  if z > 0,  α(ez − 1)  if z ≤ 0 }

ELU smoothly approaches −α for large negative inputs, keeping the mean activation closer to zero (which helps training). More computationally expensive than ReLU due to the exponential.

GELU (Gaussian Error Linear Unit)

GELU(z) = z · Φ(z)     where Φ(z) is the CDF of the standard normal distribution

GELU weights inputs by their probability of being positive under a Gaussian distribution. Used in modern architectures like GPT and BERT. It's a smooth approximation of ReLU that allows small negative values through.

When should I use Leaky ReLU vs standard ReLU?

Start with standard ReLU — it's the default for a reason and works well in most cases. Switch to Leaky ReLU if you notice significant neuron death (you can monitor the fraction of zero activations). Leaky ReLU adds minimal overhead and ensures no neuron is permanently dead. In practice, the difference is often small, but Leaky ReLU is a safe fallback when training is unstable or the network seems to have reduced capacity.

Key Takeaway: Dying ReLU = neurons stuck in negative region with gradient = 0, never updating. Caused by large biases, high learning rates, or poor init. Solutions: Leaky ReLU (small slope for z < 0), ELU (smooth exponential for z < 0), GELU (probabilistic gating).
Recap: Dying ReLU: neurons permanently output 0 because z < 0 for all inputs → gradient = 0 → no updates. Causes: bad init, high LR, large biases. Fixes: Leaky ReLU (αz for z < 0), ELU, GELU. Monitor fraction of dead neurons during training.

8.6 Softmax for Multi-Class Classification

So far we've discussed hidden layer activations (sigmoid, ReLU). Now let's look at output layer activations — specifically, how to handle tasks with more than two classes.

From Binary to K Classes

In binary classification, we used a single sigmoid output neuron producing a probability p(y=1|x). But what if we have K classes? We need K output neurons, each representing the probability of its class. The problem: K raw outputs (logits) z₁, z₂, ..., zK aren't probabilities — they can be negative, they don't sum to 1, and they're hard to interpret.

Analogy: Imagine you're judging a 4-way cooking competition. Each chef gives you a "raw score" (the logit): Chef A: 2.0, Chef B: 1.0, Chef C: 0.5, Chef D: −1.0. These raw scores aren't probabilities — they don't sum to 1 and Chef D's negative score doesn't mean "negative probability." Softmax converts these into proper probabilities: [0.488, 0.180, 0.109, 0.022]. Now they sum to 1, and Chef A has a 48.8% chance of winning.

The Softmax Function

softmax(zk) = ezk / Σj=1..K ezj

For each class k: exponentiate the logit zk, then divide by the sum of all exponentials. This produces a valid probability distribution.

Properties of Softmax

PropertyWhat It MeansWhy It Matters
Valid probability distributionAll outputs ∈ (0, 1) and sum to exactly 1We can interpret outputs as class probabilities
Translation invariantsoftmax(z + c) = softmax(z) for any constant cAdding the same value to all logits doesn't change the output — only relative differences matter
DifferentiableSmooth function with well-defined gradientsEnables backpropagation through the output layer
MonotonicHigher logit → higher probabilityPreserves the ranking of logits

Visual Example: 4-Class Output Layer

Worked Example: Softmax on 4-Class Output
1 Raw logits from the network
z = [2.0, 1.0, 0.5, −1.0]
2 Exponentiate each logit
e² = 7.389, e¹ = 2.718, e⁰·⁵ = 1.649, e⁻¹ = 0.368
3 Sum of exponentials
7.389 + 2.718 + 1.649 + 0.368 = 12.124
4 Divide each by the sum
P(class 0) = 7.389/12.124 = 0.609

✓ Verification
0.609 + 0.224 + 0.136 + 0.030 = 0.999 ≈ 1.0 ✓ (rounding)

Numerical Stability Trick

In practice, computing ezk for large zk can overflow (e1000 = ∞). The solution: subtract the maximum logit before exponentiating:

softmax(zk) = ezk − max(z) / Σj ezj − max(z)

This doesn't change the output (translation invariance) but prevents overflow. Every deep learning framework implements this automatically.

Critical Rule: NEVER use softmax in hidden layers. Softmax is designed specifically for output layers in multi-class classification. Using it in hidden layers destroys the network's ability to learn because it forces all outputs to sum to 1 (competing outputs) and produces gradients that behave unpredictably. Use ReLU (or variants) in hidden layers; use softmax only at the output.
Key Takeaway: Softmax converts K raw logits into a valid probability distribution: softmax(zk) = ezk/Σezj. Properties: sums to 1, all values in (0,1), differentiable, translation invariant. Use softmax at the output layer for multi-class classification. NEVER use it in hidden layers.
Recap: Softmax(zk) = ezk/Σezj. Produces valid probabilities from logits. Used for K-class output layers. Never in hidden layers. Subtract max(z) for numerical stability. Exponentiation amplifies differences between logits.

8.7 Softmax + Categorical Cross-Entropy Loss

Softmax produces probabilities. But how do we measure how wrong those probabilities are? We need a loss function. The natural pairing for softmax is categorical cross-entropy (CCE).

Binary Cross-Entropy (BCE) Recap

For binary classification with sigmoid output (single neuron, output p = σ(z)):

BCE = −[y · log(p) + (1 − y) · log(1 − p)]

Where y ∈ {0, 1} is the true label and p ∈ (0, 1) is the predicted probability. This loss is 0 when the prediction is perfect, and → ∞ when the prediction is confidently wrong.

Why log? The logarithm penalizes confident wrong predictions much more than incorrect predictions. If y = 1 and p = 0.9, the loss is −log(0.9) ≈ 0.105 (small). If y = 1 and p = 0.01, the loss is −log(0.01) ≈ 4.605 (large). This encourages the model to be both correct AND confident.

Categorical Cross-Entropy (CCE)

For multi-class classification with K softmax output neurons (probabilities p₁, p₂, ..., pK):

CCE = −Σk=1..K yk · log(pk)

Where yk is 1 for the true class and 0 for all others (one-hot encoding). This simplifies to:

CCE = −log(ptrue class)

The loss is simply the negative log probability assigned to the correct class.

Worked Example: CCE Loss
1 True label: class 2 (one-hot: [0, 0, 1, 0])
The true class is class 2 out of 4 classes.
2 Softmax output: [0.1, 0.2, 0.6, 0.1]
The model assigns 60% probability to class 2.
3 CCE = −log(0.6) = 0.511
Only the true class contributes: CCE = −(0·log(0.1) + 0·log(0.2) + 1·log(0.6) + 0·log(0.1)) = −log(0.6)
4 If the model were more confident: p₂ = 0.9
CCE = −log(0.9) = 0.105 — much lower loss for a better prediction.
5 If the model were wrong: p₂ = 0.01
CCE = −log(0.01) = 4.605 — high penalty for confident wrong prediction.

Why Softmax + CCE Works Well Together

When you combine softmax output with CCE loss, the gradient has a remarkably clean form:

∂L/∂zk = pk − yk

This is the predicted probability minus the true label. For the correct class, if p < 1, the gradient is negative (pushing z up). For incorrect classes, if p > 0, the gradient is positive (pushing z down). The gradient is intuitive and well-behaved.

Don't mix and match carelessly: Softmax + CCE is a matched pair. Sigmoid + BCE is another matched pair. Using sigmoid + CCE or softmax + BCE will produce incorrect gradients and poor training. Always pair them correctly (see Decision Guide in Section 8).

Output Layer Comparison

Key Takeaway: CCE = −Σyk·log(pk) = −log(ptrue class). Measures how much the true class probability deviates from 1. Pair softmax + CCE for multi-class, sigmoid + BCE for binary. The combined gradient is simply p − y, which is clean and intuitive.
Recap: CCE loss for multi-class = −log(ptrue). Paired with softmax output. BCE for binary = −[y·log(p) + (1−y)·log(1−p)], paired with sigmoid. Gradient of softmax+CCE is p − y. Don't mix activation/loss pairs.

8.8 Decision Guide: Choosing Output Activation & Loss

One of the most common exam mistakes is mixing up activation functions and loss functions. This section gives you a clear decision framework.

Task Type Output Neurons Output Activation Loss Function Example
Binary Classification 1 Sigmoid Binary Cross-Entropy (BCE) Spam vs. not spam
Multi-class (mutually exclusive) K Softmax Categorical Cross-Entropy (CCE) Digit recognition (0–9)
Multi-label (non-exclusive) K K independent Sigmoids BCE per output (mean) Image tagging (can be multiple)
Regression 1 (or K) Linear (none) MSE or MAE Predicting house price
Analogy: Think of it like ordering at a restaurant. Binary classification = "Would you like dessert? Yes/No" (one question, one answer). Multi-class = "Which of these 5 desserts would you like? Pick exactly one" (mutually exclusive). Multi-label = "Which of these toppings would you like? Check all that apply" (non-exclusive). Regression = "How much would you like to pay?" (continuous number). Each question type needs a different scoring system (loss function).
Multi-class vs Multi-label — the key distinction:
Multi-class: Exactly one class is correct. The classes are mutually exclusive (a digit is either 3 or 7, never both). Use softmax (outputs sum to 1, so choosing one class reduces the probability of others).

Multi-label: Multiple classes can be correct simultaneously. An image can contain both a cat and a tree. Use independent sigmoids (each output is an independent probability, no competition between outputs).

Decision Flowchart

How to Choose: Step by Step
1 Is the output a continuous number?
YES → Regression: Linear output + MSE/MAE loss
NO → Go to step 2
2 How many classes?
2 → Binary: Sigmoid output (1 neuron) + BCE loss
More than 2 → Go to step 3
3 Are classes mutually exclusive?
YES (exactly one correct) → Softmax (K neurons) + CCE loss
NO (multiple correct) → K Sigmoids + BCE per output
Why can't I use sigmoid for multi-class?

You technically can — and some older systems did. But sigmoid for K outputs treats each class independently. The K outputs don't sum to 1, so they aren't probabilities (they could sum to 1.5 or 0.3). This means: (1) you can't interpret them as a proper probability distribution, (2) the model can be simultaneously very confident about multiple conflicting classes, and (3) the gradient doesn't encourage competition between classes. Softmax forces the outputs to compete (higher probability for one means lower for others), which naturally fits mutually exclusive classification.

Common Exam Mistake: Using sigmoid with 10 output neurons for digit classification. This is wrong because: (1) sigmoid outputs are independent — they don't sum to 1, so they aren't a valid probability distribution. (2) You should use softmax with K=10 outputs + CCE loss. (3) Or if you insist on sigmoid, you'd need to manually normalise the outputs, which is non-standard and error-prone.
Key Takeaway: Binary → Sigmoid + BCE + 1 output. Multi-class → Softmax + CCE + K outputs. Multi-label → K Sigmoids + BCE per output. Regression → Linear + MSE/MAE. Always pair the correct activation and loss function.
Recap: Decision guide: (1) Regression → Linear + MSE. (2) Binary → Sigmoid + BCE. (3) Multi-class → Softmax + CCE. (4) Multi-label → K Sigmoids + BCE each. Never mix mismatched pairs.

8.9 Overfitting in Deep Networks

Deep neural networks have enormous capacity — they can learn incredibly complex patterns. But this capacity is a double-edged sword: they can also memorise training data instead of learning generalisable features.

Why Deep Networks Overfit Easily

A deep network with millions of parameters can fit the training data perfectly — even if the labels are random noise. The more parameters relative to the amount of training data, the更容易 the network memorises rather than generalises.

Analogy: Imagine a student preparing for an exam. A good student understands the concepts (generalises) and can answer new questions they've never seen. A bad student memorises the exact answers to the practice questions (overfits) but can't handle any variation. A deep network with too many parameters is like giving the student the answer key — they'll ace the practice test (training loss → 0) but fail the real exam (validation loss → ∞).

The Capacity-Generalisation Trade-off

More capacity (layers, neurons, parameters) = ability to learn more complex patterns. But also = higher risk of memorising noise. The goal is to find the sweet spot: enough capacity to capture the true pattern, but not so much that the model memorises the noise.

Warning: Overfitting doesn't mean the model is "bad" — it means the model has learned the training data too well, including its noise and idiosyncrasies. A model that overfits is actually very powerful — it just needs to be constrained (regularised) to focus on the signal rather than the noise.
Can a model underfit AND overfit at the same time?

Not simultaneously on the same data, but different parts of the model can exhibit different behaviors. For example, early layers might underfit (not learning useful low-level features) while later layers overfit (memorising high-level patterns). This is one reason why different regularisation strategies are applied to different parts of the network — e.g., dropout on later layers, batch norm throughout.

Key Takeaway: Deep networks have enormous capacity, making them prone to overfitting (memorising training data). Signs: training loss ↓ while validation loss ↑, large weights, confident wrong predictions. The solution is regularisation — constraining the model to learn generalisable patterns.
Recap: Overfitting = model memorises training data, fails on new data. Signs: train loss ↓, val loss ↑; large weights; confident wrong predictions. Deep networks overfit easily due to high capacity. Regularisation is the solution.

8.10 Regularisation Techniques

Regularisation is any technique that constrains the model to reduce overfitting without significantly reducing its ability to learn. Here are the most important techniques for deep networks.

1. L2 Regularisation (Ridge / Weight Decay)

Add a penalty proportional to the square of each weight to the loss function:

Lregularised = Loriginal + λ · Σwi²

The λ (lambda) hyperparameter controls the strength. L2 penalises large weights, pushing them toward zero. This discourages the model from relying too heavily on any single feature.

Why L2 helps: Large weights mean the model is very sensitive to small changes in input features — a hallmark of overfitting. By penalising large weights, L2 forces the model to distribute its reliance across many features, producing smoother decision boundaries that generalise better. In gradient descent, L2 adds −2λw to each weight gradient, effectively pulling weights toward 0 at every step (hence "weight decay").

2. L1 Regularisation (Lasso)

Add a penalty proportional to the absolute value of each weight:

Lregularised = Loriginal + λ · Σ|wi|

L1 pushes many weights to exactly zero (not just small), effectively performing feature selection. This produces sparse models that are easier to interpret.

L2 (Ridge)L1 (Lasso)
Penaltyλ · Σw²λ · Σ|w|
Effect on weightsShrinks toward 0 (rarely exactly 0)Pushes to exactly 0 (sparse)
Feature selectionNo — keeps all featuresYes — zeros out irrelevant features
Best forWhen all features are usefulWhen many features are irrelevant

3. Dropout

During each training step, randomly "drop" (set to 0) a fraction p of neurons. Typically p = 0.2 to 0.5.

During training: each neuron is "off" with probability p

During inference: all neurons active (but outputs scaled by 1 − p)

Analogy: Imagine a team working on a project, but each day, some team members call in sick randomly. The team can't rely on any specific person always being there, so everyone learns to be more versatile and the team develops redundant capabilities. At test time, everyone shows up, and the team is stronger because each member is more capable. Dropout forces the same principle: neurons can't rely on specific other neurons always being active, so each neuron learns more robust features.
Why dropout works:
1. Prevents co-adaptation: Without dropout, neurons can specialise together (neuron A only works when neuron B is also active). Dropout forces each neuron to learn features that are useful independently.
2. Implicit ensemble: Each training step uses a different sub-network. At test time, using all neurons approximates averaging the predictions of exponentially many sub-networks — an implicit ensemble of 2N models (where N is the number of neurons).
Warning: Dropout is only active during training. During evaluation/testing, ALL neurons must be active. Forgetting to set the model to eval mode (e.g., model.eval() in PyTorch) keeps dropout active during inference, producing random, degraded predictions. This is one of the most common debugging mistakes in deep learning.

4. Early Stopping

Monitor validation loss during training. Stop when validation loss stops decreasing (or starts increasing), even if training loss is still falling.

Stop training when val_loss has not improved for "patience" epochs

The patience parameter (e.g., 5, 10, 20) determines how many epochs to wait for improvement before stopping. This prevents the model from training long enough to overfit.

Why is early stopping the simplest regulariser?

Early stopping requires no changes to the model architecture or loss function. You just monitor the validation loss and stop training when it starts to rise. It's like knowing when to stop studying — continuing to review the same material won't help and might give you false confidence. The patience parameter adds robustness: small fluctuations in validation loss won't trigger premature stopping. Early stopping is almost always worth using, even alongside other regularisation techniques.

5. Batch Normalisation

Normalise the activations of each layer to have zero mean and unit variance within each mini-batch:

ẑ = (z − μbatch) / √(σ²batch + ε)     then     z̃ = γ · ẑ + β

Where γ and β are learnable parameters (scale and shift) that allow the network to undo the normalisation if needed.

Why batch norm helps:
1. Reduces internal covariate shift: Each layer receives inputs that change distribution as earlier layers update. Batch norm stabilises these distributions.
2. Smooths the loss landscape: Gradients become more predictable, allowing higher learning rates and faster convergence.
3. Mild regularisation effect: Since normalisation is computed per mini-batch, there's inherent noise from the batch statistics, which acts as a regulariser similar to dropout.

6. Data Augmentation

Create additional training examples by applying realistic transformations to existing data:

Data TypeAugmentation Techniques
ImagesRandom rotation, flip, crop, colour jitter, cutout, mixup
AudioAdd noise, time stretch, pitch shift, time shift
TextSynonym replacement, random insertion/deletion, back-translation
TabularSMOTE (synthetic minority oversampling), noise injection
Key Takeaway: Regularisation prevents overfitting. Key techniques: L2 (penalises large weights), L1 (penalises large weights, creates sparsity), Dropout (randomly zeros neurons, prevents co-adaptation), Early Stopping (stop when val loss rises), Batch Norm (normalises activations, smooths loss landscape), Data Augmentation (creates synthetic training data).
Recap: L2: +λΣw², weight decay. L1: +λΣ|w|, sparse models. Dropout: random p fraction off during training, all on at test. Early stopping: monitor val loss, patience. Batch norm: normalise per batch. Data augmentation: synthetic transforms.

In practice, you don't choose just one regularisation technique — you combine several. Each addresses a different aspect of overfitting.

8.12 Complete Network Design Checklist

When designing a neural network, there are many decisions to make. Here's a comprehensive checklist that covers architecture, activation, loss, regularisation, and training.

ComponentDecisionRecommendation
Hidden layer activation Which activation for hidden layers? ReLU (default). Leaky ReLU if dead neurons detected. GELU for transformers.
Output activation Which activation for output layer? Task-dependent: Sigmoid (binary), Softmax (multi-class), Linear (regression).
Loss function Which loss to use? Must pair correctly: BCE (sigmoid), CCE (softmax), MSE/MAE (linear).
Regularisation baseline What regularisation to start with? Early stopping (patience=10) + L2 weight decay (λ=1e-4).
Normalisation Should I use batch norm? Yes, after each linear layer and before activation. Improves training stability.
Weight initialisation How to initialise weights? He init for ReLU (W ~ N(0, √(2/nin))). Xavier/Glorot for tanh/sigmoid.
Optimiser Which optimiser? Adam (default). SGD+momentum if you need best generalisation and have time to tune LR.
Monitoring What to track during training? Both training AND validation loss curves. The gap indicates overfitting.
Why He init for ReLU and Xavier for tanh?

He initialisation (W ~ N(0, 2/nin)) ensures that the variance of activations is preserved through ReLU layers. Since ReLU zeros out ~50% of neurons, we need the remaining ones to have twice the variance to compensate — hence the factor of 2. Xavier initialisation (W ~ N(0, 2/(nin + nout))) is designed for tanh/sigmoid, which preserve variance differently because they're symmetric and don't zero out activations. Using the wrong initialisation can cause activations to explode or vanish even before training begins.

8.13 Common Pitfalls

These are the mistakes that students (and practitioners) make most frequently. Understanding them will save you hours of debugging.

Pitfall 1: Softmax on a hidden layer
Using softmax as the activation in a hidden layer forces all outputs to sum to 1, making them compete. This destroys the network's ability to learn distributed representations. Hidden layers should use ReLU (or variants). Softmax is ONLY for the output layer in multi-class classification.
Pitfall 2: Heavy dropout on small networks
Applying dropout with p = 0.5 to a network with only 32 neurons in a hidden layer means, on average, only 16 neurons are active per training step. This can cripple the network's capacity, causing underfitting. Reserve heavy dropout for large networks (256+ neurons). For small networks, use lighter dropout (p = 0.1–0.2) or rely on other regularisers.
Pitfall 3: Sigmoid on a 10-class output
Using sigmoid with 10 output neurons for digit classification treats each class independently. The outputs don't sum to 1, so they aren't probabilities. Use softmax with K=10 outputs for mutually exclusive multi-class classification, paired with CCE loss.
Pitfall 4: ReLU on a regression output
Using ReLU as the output activation for regression restricts predictions to ≥ 0. If your target values can be negative (e.g., temperature in Celsius, financial returns), ReLU will never predict them. Use a linear (identity) output for regression.
Pitfall 5: Forgetting train/eval mode with dropout
Dropout randomly zeros neurons during training. If you forget to switch the model to eval mode (model.eval() in PyTorch, model.training = False in TensorFlow), dropout remains active during inference. This produces random, degraded predictions. Always set eval mode before testing and train mode before training.
Pitfall 6: L2 with sigmoid output and no batch norm
L2 weight decay penalises large weights. But sigmoid outputs are computed as σ(Wx + b), where the scale of W directly affects how "sharp" the sigmoid is. Heavy L2 can flatten the sigmoid, making all outputs ≈ 0.5 — the model loses discriminative power. Batch norm after the linear layer but before sigmoid helps maintain proper scaling. Alternatively, use lighter L2 on the output layer.
How do I debug a network that's not learning?

Check these in order: (1) Is the loss decreasing at all? If not, the learning rate might be too high or too low. (2) Are the gradients reasonable? Check for NaN or very large values (exploding gradients). (3) Are most activations zero? Check for dying ReLU (too many dead neurons). (4) Is the output activation correct for the task? (5) Is the loss function paired correctly with the output activation? (6) Are you in the right train/eval mode? (7) Is the data normalised? Neural networks are sensitive to input scale — inputs should typically be standardised (zero mean, unit variance).

Key Takeaway: Common pitfalls: softmax in hidden layers, heavy dropout on small nets, sigmoid for multi-class output, ReLU for regression output, forgetting train/eval mode with dropout, L2 with sigmoid without batch norm. Always verify: activation choice, loss pairing, train/eval mode, and data normalisation.
Recap: Top 6 pitfalls: (1) Softmax in hidden layers, (2) Heavy dropout on small nets, (3) Sigmoid for 10-class, (4) ReLU on regression output, (5) Forgetting eval mode, (6) L2 + sigmoid without batch norm. Debug methodically: check loss, gradients, activations, output activation, loss pairing, modes, normalisation.

Flashcards

What happens to a deep network without activation functions?
It collapses to a single linear transformation: ŷ = W'x + b'. No matter how many layers you stack, the composition of linear functions is still linear. Activations are the ONLY source of expressive power.
What is the maximum value of the sigmoid derivative σ'(z)?
0.25 (at z = 0). For all other values, σ'(z) < 0.25. This means gradients shrink by at least 75% at each sigmoid layer, causing vanishing gradients in deep networks.
Define ReLU and its derivative.
ReLU(z) = max(0, z). Derivative: 1 if z > 0, 0 if z < 0, undefined (use 0) at z = 0. Gradient = 1 for active neurons means no vanishing — the key advantage over sigmoid.
What is the "dying ReLU" problem?
When a ReLU neuron's weighted sum is negative for ALL training examples, it outputs 0 and has gradient 0 — so its weights never update. The neuron is permanently dead. Caused by large negative biases, high learning rates, or poor initialisation.
Write the softmax formula and state its key properties.
softmax(zk) = ezk / Σj ezj. Properties: (1) outputs sum to 1 (valid probability distribution), (2) all outputs in (0,1), (3) translation invariant (softmax(z+c) = softmax(z)), (4) differentiable. Use for multi-class output layers only.
What is categorical cross-entropy (CCE) loss?
CCE = −Σk yk · log(pk) = −log(ptrue class). Measures how wrong the predicted probability for the true class is. Paired with softmax output. Combined gradient: ∂L/∂zk = pk − yk.
Which activation/loss pair is correct for binary classification?
Sigmoid output (1 neuron) + Binary Cross-Entropy (BCE) loss. The sigmoid maps the logit to (0,1), and BCE measures the deviation from the binary label. Don't use softmax for binary — it's equivalent but wasteful.
How does dropout prevent overfitting?
Two mechanisms: (1) Prevents co-adaptation — neurons can't rely on specific other neurons always being active, so each learns more robust features. (2) Implicit ensemble — each training step uses a different sub-network, approximating the average of 2N models at test time.
What is the difference between L1 and L2 regularisation?
L2 (Ridge): penalty = λΣw², shrinks weights toward 0 but rarely to exactly 0. Keeps all features. L1 (Lasso): penalty = λΣ|w|, pushes many weights to exactly 0, performing feature selection. L1 produces sparse models; L2 produces small but non-zero weights.
Why is batch normalisation helpful?
Batch norm: (1) Reduces internal covariate shift — stabilises the distribution of inputs to each layer. (2) Smooths the loss landscape — allows higher learning rates and faster convergence. (3) Provides mild regularisation — batch statistics add noise during training.
Which weight initialisation should you use with ReLU?
He initialisation: W ~ N(0, √(2/nin)). The factor of 2 compensates for ReLU zeroing out ~50% of neurons, preserving activation variance through layers. Use Xavier/Glorot init for tanh or sigmoid activations instead.
What are the signs of overfitting in a neural network?
(1) Training loss decreases while validation loss increases. (2) Large gap between training and validation accuracy. (3) Large weight magnitudes. (4) Confident wrong predictions (high probability for incorrect class). The diverging loss curves are the most reliable indicator.
What is the translation invariance property of softmax?
softmax(z + c) = softmax(z) for any constant c. Adding the same value to all logits doesn't change the output probabilities. This is why we can subtract max(z) before computing softmax for numerical stability without affecting the result.
